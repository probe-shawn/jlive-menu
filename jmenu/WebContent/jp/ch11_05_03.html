Buffer データソースへテキストファイルのロード (load text file to buffered data source)

　

  • テキスト

    　

    • ファイル URL  (file URL)： テキストファイルのパス。

    • Tab Delimited ： 選択すると (checked)，Tab 記号で分割されたテキストになります。選択しない (unchecked)と ，固定長さ (Fixed length)のテキストになります。

    • テキストレコード定義 (define text record)： テキストレコードの定義。

      固定長さ (Fixed length)のテキストの場合，正確にレコード BYTE 数を設定しなければなりません。Tab 記号で分割されたテキストの場合，システムは長さプロパティを無視します。

      • 上へ ： 上へ移り。

      • 下へ ： 下へ移り。

      • 削除 ： データフィールド定義の削除。

      • データフィールド (field)： 文字データの一時フィールド名，fld1から順序並べが出来ます。

      • タイプ (type)： データスタイル，VARCHAR、INTEGER、FLOAT、DATE、TIME、TIMESTAMP。

      • 長さ (length)： データの長さ，Byte で計算 。

      • フォーマット (format)： 日付の場合，フォーマットが指定される必要があります。システムはフォーマットでパサー (parser)が出来ます。例えば：データは"2001-12-31 14:29:51"の時， TIMESTAMP スタイルを選択し，フォーマットは："yyyy-MM-dd hh:mm:ss"にすると，システムはBufferデータソースに"20011231142951"を記入します。　

　

  • Buffer データソース ：

    　

    • Buffer データソース (buffer data source)： Buffer データソースの選択 。

    • 対照マップ (map transfer fields)： テキスト枠を Bufferデータソースに対応させます 。

      • データフィールド (field)： Buffer データソースで定義した枠順位番号。

      • 名前 ： Buffer データソース名。

      • テキストフィールド ： 前のページで設定されたテキストフィールドの選択。
