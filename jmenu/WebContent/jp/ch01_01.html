1.1.1 jLIVE Builderのインストール (Installation Notes)


jLIVE Builderは最新バージョンのJava™ 2 開發環境であるJava™ 2 SDKをサポートします， Java™2 SDK は Sun Microsystems(TM), Inc. 社の商品です。関連情報は Sun Microsystems(TM), Inc. 社のホームページで説明されています 。

インストールパスには空白スペースの使用は不可で，例えば : C:\Program Files\ ，JDK 中の一部分のツール(Tools)作業環境指数(environment variables) がそれによって予想つかない問題が生じるかもしれません。

jLIVE Builderのインストールが完成された後，ディレクトリーは以下のようです:

jLIVE Builderを起動する前に 以下の環境設定確認を行わなければなりません:

1.       Java™ 1.4.x SDK インストール済み，JAVA_HOME 環境変数が正確に設定されたかどうか。

2.       データベーススタートアップ。

3.       サンプルをテストするなら，先に mySQL データベースをスタートアップします。

4.       jLIVE™ middlewareミドルウェアをスタートアップします。デフォルトの middle wareはTomcat 4.0の配下で動作します。 Tomcat 4.0をスタートアップする為に，jLIVE\tomcat\bin\startup.batをクリックします。その前， JAVA_HOME と CATALINA_HOMEの環境変数が正確に設定されたかどうかを確認します。

5.       初めて jLIVE Builderを実行する時，システムはサンプルパスとインストールパスを取得します。

1.1.2. サンプル実行


jLIVE Builder™ 內のアプリケーションサンプルが、使用しているデータベースは mySQL　です。このアプリケーションはテストとデモ用に使用を限定します。 サンプルを起動するには、Java™ と jLIVE Builder™ を起動する前に mySQL　データベースを起動します 。

1.       mySQLスタート : jLIVE\mysql\bin\winmysqladmin.exe。

2.       Tomcatスタート :  jLIVE\tomcat\startup.bat。

3.       直接に(jar file) jnorth.jarを実行することができ，操作方法は一般なWindows 方式と同じです。或いはHTML file (jnorth.html) をブラウザで実行します (Browser)。

注）ｍｙＳＱＬは、使用される方が、ｍｙＳＱＬのライセンスルールに従い使用して下さい。

1.1.3 Tomcat 4.0.x 啟スタートアップ

　

Tomcat バージョン : jakarta-tomcat-4.0.1 , http://jakarta.apache.org/builds/jakarta-tomcat-4.0/release/v4.0.1/bin/

jLIVE Builder™ Tomcat パス : jLIVE\tomcat。

　

最初に Tomcatを起動する前に必ず JAVA_HOME と CATALINA_HOME の環境変数を設定してください。

環境変数 ( JAVA_HOME と CATALINA_HOME )

·         Windows NT/2000/XP

1.                               【パネル】->【システム】->【詳細】-> 環境変数。

2.                               新規システム変数，変数名称 : JAVA_HOME ，変数 : c:\j2sdk1.4.0。

3.                               新規システム変数，変数名称 : CATALINA_HOME ，変数 : c:\jLIVE\tomcat。

　

　

　

　

　

·         Windows 95/98/me

　

1.                               編集 C:\autoexec.bat。

2.                               新規 SET CATALINA_HOME=C:\jLIVE\tomcat  と  SET JAVA_HOME=C:\j2sdk1.4.0。

3.                               リスタート。

·         スタートアップとシャットダウン Tomcat (Start up and shut down the Tomcat)

　

1.                               c:\jLIVE\tomcat\bin\startup.bat。

2.                               c:\jLIVE\tomcat\bin\shutdown.bat。

　

·         Tomcat Troubleshooting

Windows上Tomcat 4.0の設定に関する諸問題 : 

1) 一番発生しやすい問題は， Tomcat HTTP のデフォルトポートPort 8080が既に他の Web server に使用されている，例えば Oracle  9i の application server。ポートを換えるには :  

$CATALINA_HOME/conf/server.xml

を開き，8080を見つけ，使用されていないポートに換えます。1024より大きいのを進めます，1024より小さかったら作業システム説明をさんしょうして 請參照各作業システム説明を参照してください。

再度Tomcatを起動すれば正常に作業が行われます。Port 8080 を換えた場合，Tomcat のURLも同じく直さなければなりません。例えば :  Port を 1977にされた場合， URL は http://localhost:1977/ 。

2)  Windows 9X/ME のバッチファイルを実行する時に "out of environment space" のエラーが出ます。 

3) localhost が存在しない ('localhost' machine isn't found)， Proxyが原因です。この場合，Proxy ホストにアクセスしないように， ブラウザの Proxy設定を換えます 。

Netscapeの場合， Edit/preferences -> Advanced/proxies。 Internet Explorerの場合，Tools -> Internet Options -> Connections -> LAN Settings。

1.1.4 JDBC ドライバ (Drivers)

　

jLIVE Builder™ はJDBC バージョン 2.0 API　をサポートします。ドライバのプロバイタと対応データベースシステムにつき，http://industry.java.sun.com/products/jdbc/drivers　を参考してください。 jLIVE Builder™ は JDBC TYPE 1, 2, 3, 4 ドライブ方式をサポートします。 TYPE 1をサポートしますので ，JDBC-ODBC bridgeを利用でき，ODBCを経由でデータベースにアクセスします。しかし，JDBC-ODBC bridge の接続方式は効率が非常によくない為，Java™ はデータベース接続テストにしか推薦できません。JDBC-ODBC bridgeを使わざる得ない時に ， http://www.merant.comを参考してください， Merant 社が提供した MERANT ODBC Driver Packを使用すれば，JDBC-ODBC bridge の效率が大幅にアップできます。


MSSQL Server の場合， http://www.inetsoftware.de/ (i-NET Software company)を参考してください，試用バージョン MSSQL Server　ドライブ (TYPE 4)をダウンロードして使えば，JDBC-ODBC bridge よりかなりよい。

　

ORACLE の場合， JDBC Driver (classes12.jar) を Tomcat\libの中に置いておき， システムがドライバを利用できるようにClassPath を設定します。関連JDBC Driverの設定及び実装につき，Oracleマニュアルを参考してください。Oracle 8i (8.16) JDBC Driver として Database のMetadata　を使わないよう， 8.17バージョンを使ってください。Oracle 9i JDBC Driver の場合，必ず unpack した後， JDBC Driver 部分だけを使うように，他の Java class部分につき， Java バージョンが相違するか，交換性がないかの為，ドライバ (Driver)が導入できない場合もあります。

　

jLIVE Builder™ にドライバを実装する時に，必ず詳しくプロバイター資料を読んでください，正確にインストールするようにデータベース接続用の Database URL format String も正しく設定されなければなりません。

ドライバ使用例 : '*' は説明で入力する必要がありません。
　

·          JDBC-ODBC bridgeでMS ACCESS　の接続 。

　

1.                                ODBCAD32.EXE (%SystemRoot%\system32\odbcad32.exe)　を実行し， ODBC DSN　を作成します 。例えば "MYODBC"。

2.                                jLIVE middle ware を起動します(  jLIVE\tomcat\startup.batをクリック)。

3.                                jLIVE Builder™ を起動し，新規アプリをした後に，データベース新規を選択し，プロパティを設定します : 
データベース名 (DB Alias) :  MYODBCALIAS **
ドライバ (Driver) : sun.jdbc.odbc.JdbcOdbcDriver
URL : jdbc:odbc:MYODBC

*大小文字に区別あります。
**どんな別名でもよい (DB Alias)，システムはこれでデータベース式を識別します。

4.                               保存した後，テスト (データベース接続テスト)，システムはデータベースの接続状態をテストします。
　

·          mySQL の接続 

1.                mySQLを起動 : jLIVE\mysql\bin\winmysqladmin.exe。

2.                jLIVE middle ware を起動(  jLIVE\tomcat\startup.batをクリック)。

3.                jLIVE Builder™ を起動し，新規アプリをした後に，データベース新規を選択し，プロパティを設定します : 
データベース名 (DB Alias) : MySQL_DB **
ドライバ (Driver) : org.gjt.mm.mysql.Driver
URL : jdbc:mysql://HostName:3306/DatabaseName

*大小文字に区別あります。
**どんな別名でもよい (DB Alias)，システムはこれでデータベース式を識別します。
*** HostName : データベースのインストール先ホスト名。
**** DatabaseName : データベース名。
　

4.               保存した後，テスト (データベース接続テスト)，システムはデータベースの接続状態をテストします。
　

·         ORACLE の接続 

1.                Oracle を起動します。

2.                jLIVE middle ware　を起動します (  jLIVE\tomcat\startup.bat　をクリック)。

3.               jLIVE Builder™ を起動し，新規アプリをした後に，データベース新規を選択し，プロパティを設定します : 
データベース名 (DB Alias) : ORACLE_DB **
ドライバ (Driver) : oracle.jdbc.driver.OracleDriver
URL : jdbc:oracle:thin:@HostName:1521:DatabaseName

*大小文字に区別あります。
**どんな別名でもよい (DB Alias)，システムはこれでデータベース式を識別します。
*** HostName : データベースのインストール先ホスト名。
**** DatabaseName : データベース名。
　

4.               保存した後，テスト (データベース接続テスト)，システムはデータベースの接続状態をテストします。
　

·          inet JDBC Driver の接続 

1.               inet の指示通りに i-netSPRINTAをインストールします。

2.                jLIVE middle ware (執行  jLIVE\tomcat\startup.bat)。

3.                jLIVE Builder™ を起動し，新規アプリをした後に，データベース新規を選択し，プロパティを設定します : 
データベース名  (DB Alias) : MSINET **
ドライバ (Driver) : com.inet.tds.TdsDriver
URL : jdbc:inetdae7:HostName:1433?database=DatabaseName&charset=Cp950

*大小文字に区別あります。
**どんな別名でもよい (DB Alias)，システムはこれでデータベース式を識別します。
*** HostName : データベースのホスト名。
**** DatabaseName : データベース名。
*****日本語(MS932)。

4.               保存した後，テスト (データベース接続テスト)，システムはデータベースの接続状態をテストします。
