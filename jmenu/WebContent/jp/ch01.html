  • 高速なピジュアル環境 (Rapid Application Development framework)

高速なピジュアル環境（Rapid Applcation Development framework)によって、開発者は、設計しながら実行し、業務フローをより正確にし、設計品質の向上につなげることができます。システムは効率のよいJavaTM MVC (Model View Controller)テクノロジー を用いています。

  • 開発したアプリケーションはブラウザ起動とスタンドアロンで実行できます。 (browser enabled or standalone executable program)

ｊＬＩＶＥ BuilderTM開発されたアプリケーションはネットワークを通しブラウザ上で実行できます。開発者はサーバ上の最新版のアプリケーションだけをメンテナンスします。JavaTMclass loaderはダイナミックにタンロードとアップグレードCLASSを実行します。その為、システムは最も効率良くメンテナンスできるようになっています。

  • 多階層構造 (N-tieres architecture, pure java middleware)

多階層構造は業務用アプリに必須条件です。 lang=EN-US>ｊＬＩＶＥ　 middle実行時の環境は少なくても3階層（3-tiers）です。ｊＬＩＶＥ BuilderTM　は、ユーザ側（クライアント）（Zero installation client site）にはインストールする必要がなく、Webサーバもしくは、アプリケーションサーバーにインストールされたｊＬＩＶＥ TMｍｉｄｄｌｅｗａｒｅ はJDBCを通し、データベースサーバ（database server tier）つながります。3階層（3-tiers）eはこのように構成されています。ｊＬＩＶＥ TMｍｉｄｄｌｅｗａｒｅはpureJave　Servlet形で、各種のブラットフォームに対応し、多種類のアプリケーションサーバに適用しています。

  • 分散処理方式とインタープリタ (distributed computing with just in time formula interpreter)

JLIVE BuilderTMで開発されたシステムは分散処理方式を原則とします。システムはユーザのリクエストをユーザ、ミードル層、データベースサーバに分けて処理させます。例えば、レポート業務の場合、ユーザ側は資料のリスティングのリクエストを出し、ミードル層は コマンドをデータベースサーバに転送し、結果集（result set）をユーザ側に返します。レポ ートレイアウトに関しても、ユーザ側で即時に作成されます。分散処理方式はこのようにユーザに 対して速く対応し、サーバ側の効率をより良くします。これこそ JLIVE BuilderTM開発されたアプリが何故素早く実行できるかの要因の一つです。

  • リアルタイム・データベース操作 (the real time user interactive database operation)

JLIVE BuilderTMで開発されたアプリは単純なホームページと根本的に違います。プラウザの表面にコンポーネントがない場合、ユーザは操作すると、ユーザの background threadは即 時にリクニストを送り、middle ware database serverを経て資料が表面にあるコンポーネントに帰されます。このように単純にページ変換ではなく、全て自分のPCの上に行われているようにスムーズに感じます。

  • イベントドライブとオブジェクト設計コンセプト(the event-driven, object-oriental programming)

OOPはJLIVE BuilderTM開発環境の中心になっています。開発者はオブジェクトでアプリを構 造し、一つのオブジェクトにイベントドライブを関連させます。イベント処理はユーザロジッ クとビジネスロジックによって作られます。JLIVE BuilderTM開発環境報ので中、資料、コンポー ネント、レイアウト、レポート及びイベント、ビジュアルなオブジェクトを用いれば、素早く アプリを設計し、メンテナンスとアップグレートも簡単にできます。

  • プロ級レポートシステム (professional report system)

JLIVE BuilderTMレポートシステム同じくform designとdrag-drop式を採用しています。しかも、ユーザ画面とデータソースを共用します。その為、レイアウト上のすべてが印刷可能（printable　object）です。レポート設計の時、オブジェクトを指定の地域に置くだけで結構です。レポートシステムはmm、inchを単位にし、ネットワークを通し、どんなプリンターでも同一格式のレポートを印刷されます。リストアウトされた印刷結果には、フォーミュラで処理された結果が反映されています。
