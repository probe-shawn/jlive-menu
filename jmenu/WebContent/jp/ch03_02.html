3.2 フォームデザイン (form design)

フォームデザイン手順 (form design procedure)

  1. フォームのプロパティ設定 (form properties setting)
  2. データソース作成 (add data source)
  3. レイアウトコンポーネント (layout components)
  4. コンポーネントのプロパティ設定 (component properties setting)
  5. イベントフロー作成 (create event flow)
  6. イベントリンク (link event)
  7. デバッグ及び実行テスト (debug and execute form)

　

フォームプロパティ設定 (form properties setting)

　　【フォームプロパティ設定】キーをクリックし，フォームプロパティ設定でニーズによってフォーム表示レイアウトと作業フロー、イベントを設定します。

デザインのアイテム (design tips)

  • フォームタイトル (form title)
    デザインナーはフォーミュラ SetTitle("title") と FormName()を使用してフォームタイトルを設定します。
  • フォーム位置とサイズ (Form position and size)
    ランタイムに (Runtime) ，フォーミュラ FormPos(left, top) と FormSize(width, height) を利用してフォームの位置とサイズを変更することが可能です。【オープンフォーム】 (Open form)を選択すると，フォームのプロパティ設定ページが表示されます；フォーミュラ OpenForm("formname", "pos") を利用すれば，コンポーネントの相対位置をとフォーム中央に表示することが可能です。 OpenForm フォーミュラ参考 。
  • フォームタイプ (Form type)
    フォームの種類はメインフォーム (main)、ダイアログボックス (dialog)、内部フレーム (internal frame)    、隠しプロセス (process)。メイン (main)フォームはブラウザ (browser)で実行の場合，ブラウザ (browser) の画面に組み込まれます；スタンドアロン(stand alone) で実行する場合，メインフレーム (main frame) の中に組み込まれます。ダイアログボックス (dialog) は独立のフォーム (window)の形でブラウザ (browser) 或いはメインフレーム (main frame)の外に表示されます。内部フレーム (internal frame) のフォームはブラウザ中(browser)のアプレット大きさ或いはメインフレーム枠 (main frame)の範囲内に表示されます。全ての内部フォームにおいて，マウスでフォーム (active window)の切り替えは可能です。フォームが重なる場合がありますが，作業処理に影響はありません。隠しプロセス (process)フォームの場合，ユーザの操作は必要なくフォームとして表示されません，フローに準じて実行されます：データソースオープン前イベント (before data source open event) 、データソースオープン (open data source)、データソースオープン後イベント (after data source event)、フォームクローズ前イベント (final event)、フォームクローズ(invisible)。

　

データソース作成 (add data source)

データソース (data source) はアプリ (application)とモジュール (form) プロセス操作のために，データテーブルに連結する資料構築 (table structure data)です。データソース (data source)は源によって三種類に分けられています：テーブルデータソース (Table data source)、SQL データソース (SQL data source)、Buffer データソース (Buffer data source)，デザインナーはデータソースによる設定に基づいてフォームにデータソースを追加します。

デザイン提示 (design tips)

　

レイアウトコンポーネント (layout components)

レイアウトコンポーネントにつき，１．ツールバーから選択 (drag & drop toolbar components)，２．データフィールドからドラッグ＆ドロップ (drag & drop data field)，３．データソースからドラッグ＆ドロップ (drag & drop data source)。 レイアウトコンポーネント (layout component)、コンポーネントセレクト、移動、編集 (select、move、edit components)、コンポーネント整列 (alignment)、コンポーネント複写、コピー、貼り付け (duplicate、copy、paste)。

デザイン提示 (design tips)

  • 整列 (alignment)
    タブコンポーネント (Tabbed component) は整列をサポートしません。
  • コンポーネント位置 (position)
    一つか多数のコンポーネントを選択し，マウスかキー Ctrl 加 ←↑↓→ で位置を微調整します。

　

コンポーネントプロパティ設定 (component properties setting)

デザイン提示 (design tips)

  • デザインタイム (design time)

    • 共用プロパティ設定 (common properties)：フォーマット (format)、 外枠 (border) 、フォント (font)。

    • データ連結設定 (data binding) ：データフィールド(data field)、フォーミュラ (formula) 。

    • 特殊プロパティ設定 (extra properties)：コンポーネントの特殊プロパティ。

    • 起動イベント設定 (fire event)：コンポーネント種類によってイベント起動 (action)、変化イベント (change)、 GainFocusイベント、LostFocus イベント、ダブルクリック (double click)、右キーイベント (right click)。

  • ランタイム (run time)

    • コンポーネント位置調整 (position)、サイズ (size)、可視 (visible)、使可能 (enable)、フォント (font)、前景カラー(foreground and background color)

      • イベント起動 (fire event)方式：コンポーネントイベント (component event) の位置 (position)、サイズ(size)、可視 (visible)、使可能 (enable)、フォント (font)、前景カラー (foreground and background color) イベント。

      • フォーミュラ (formula)方式：SetProp("コンポーネント名", "プロパティ", "值") ， Ger-Set Component Propertiesコンポーネント参考。

    • コンポーネントFocusの調整

      • イベント (fire event)起動方式：起動Focusイベント (component event) 。

      • フォーミュラ (formula) 方式：SetProp(ComponentName, "focus", "1")。

    • コンポーネント調整

      • 起動イベント (fire event)方式：起動コンポーネントイベント (component event) 。

      • フォーミュラ (formula)方式：SetProp("コンポーネント名", "プロパティ", "值") ，Ger-Set Component Propertiesコンポーネント参考。

　

　

イベントフロー作成 (create event flow)

イベントフロースキーマ (event flow schema)はデザインナーのため新規 (create)、編集 (edit)、ビュー(view) イベント (event object)，分析 (analysis)、企画 (route) 、調整 (adjust) イベントフロー (event flow)及び実行テストとイベント中断 (interrupt)の設定を提供します。 新規イベント (create new event)、編集、削除、複写、コピー、イベント貼り付け (edit, delete, duplicate, copy, paste  event)。

　

　リンクイベント (link event)

イベントフロー (event flow)はユーザとシステム (system flow) によって駆動されます。コンポーネントとデータソース設計を行う時に，ユーザの行為に基づいてリンク (link) イベント，イベントフロー (event flow)を起動し，ユーザへ作業の反応 (respond) を行います；システムによるイベントの起動は，例えば：フォームオープン最中，順序によってデータソースオープン前イベント (before data source open event)及びデータソースオープン後イベント(after data source open event)，フォーム閉じる直前，最終イベント (final event)起動。

　

フォームデバッグと実行 (debug and execute form)

jLIVE Builder™ はオブジェクト指向 (OOP) とイベントドライブ (event-driven) がアプリケーション開発の中核コンセプト (concept)であり，オブジェクトとイベントが jLIVE Builder™ のデバッグ(bug free)で処理される時に ，エラーがあれば，イベントルート (event route)、システム変数 (system variable)、プロセスデータ配送 (computed and transferred data)の部分に生じます。そのため、 jLIVE Builder™ はフォームデザイン段階，イベント中断 (event interrupt) 機能を提供し、システムパラメーター、プロセスデータ配送において，デザインナーによる監視及び修正が可能です。

デザイナーはイベントフローの中，イベント実行前後に中断ポイントを設定します；実行テストの時，トラブルシューティングしやすい為，一時にこの中断ポイントを取り消します。
