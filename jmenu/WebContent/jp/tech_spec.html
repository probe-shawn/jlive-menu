Technical Specification

This is a quick guide to the technical and user features of the jLIVE Builder™ development system and the jLIVE™ application server (middle ware). This document avoids lengthy explanations and general background about Web architecture.

jLIVE Builder™

Solution for developing, deploying and managing n-tier internet and intranet application, support report system for printing anywhere on any platform. 

A system for both the development and deployment of intranet and internet business applications, generally built over existing databases. It is not just another IDE. jLIVE Builder™ gives you a way to quickly make interactive database application for Java and deploy them over the Web -- all in one comprehensive package. 

jLIVE Builder™ General Features

  • Build Java applications, applets accessible for execution by any client, on any platform, world wide.

  • N-tiers system.

  • Highly visual IDE design & report framework.

  • Visual Java designers and wizards for drag-and-drop Java™ JFC/Swing application and report development.

  • Integrated form design and report design system.

  • Report preview and printing computed by client.

  • Virtual data-stream mode, request on demand.

  • Make truly interactive user interface and distributed database computing application.

  • Heterogeneous database manipulate simultaneously.

  • Object oriental programming and event-driven concept.

  • XML repository for programming logic and flowchart.

  • Generate Java™ source code for advance user.

  • Build-in Html converter for automatic plug-in JRE (Java Runtime Environment).

  • Redirect JRE-plugin cab location supported for intranet and internet deployment.

  • Preload-image strategy accelerate loading time.

  • Visual development using the Java 2 JFC/Swing components for professional UI development in Java.

  • High-performance data-aware components with MVC architecture inside.

  • Security and authorization module supported, and many roles for user can be set.

  • Security and authorization creation, update, maintain modules source code included.

  • E-mail support.

  • Bar code support.

  • Workgroup design version, supported multiple developer design simultaneously. 

  • Online update deploy modules.

  • Support for the latest Java standards.

Form Design Framework

jLIVE Builder™ follow the object oriental programming and event-driven concept to make applications. First, specify the data source that will be manipulated within the form;  layout data-aware components to control data flow;  route the user interactive events to complete process logic; then set the form properties and let it be called by other form or menu.

Formula Design Pane

The Formula is a high performance interpreter expression, support for computing data and control program process flow. there are over 100 functions divided into Math, Compare, Logic, String, Date, Function, Components, Variable, DsOperation, Flow and  Special categories. 

  • The Data Source Fields, components, System variables and Globe and Local Hashtable can be used as formula operand.

  • The Data Source meta-information such as Record Number, EOF, BOF, IsEmpty, Insertion Mode, Update Mode, Total Record and Read Only can be used within formula computing.

  • The formula and functions will return a string value always, that could be a computing result or a process function return state. If that be used as flow condition, the empty string or "0" value mean to FALSE state, otherwise was TRUE.

  • The IF-THEN-ELSE is nested expression.

  • The parameter of Formula function, could be another expression.

Event Object and Schema

Event is a object that will be hooked on components or system. Event respond the user and system action to complete the process flow. For example, you can have a changed-event to handle the Edit component when user key in; a initial-event when Form initialization; a fetch-event when data source got next record.

According to specified condition, event can link to next-event or otherwise-event, therefore a series of events will be executed to complete the mission. there is a special reentry constraint for event to control the endless loop.

Event Object

  • Event Name -- identified event

  • Condition Formula -- branch the event flow

  • Otherwise Event -- do when condition formula return FALSE

  • Perform -- the mission this event will do

  • Reentry Constraint -- Prevent endless loop

  • Next Event

 Event Schema framework

Event schema framework provide the developer to create and route event flow. Using the event map, developer can overview the process flows which invoked by user and system, reroute or break it or show a message box to validate internal data.

  • Event Name -- Name of event

  • Event Condition Pane -- jLIVE™ formula viewer 

  • Otherwise Event -- Do when condition formula return false 

  • Perform Pane -- The detail parameter of event perform

  • Next Event -- Next event

 

jLIVE Builder Report System

jLIVE™ report system is highly integrated with Form design framework, it support the same visual RAD environment as Form design, drag and drop printable object to layout report. The printable content is dynamic computing by jLIVE™ formula, so all the components content and data source can be printed. 

The general report features :

  • Print or preview from browser on any client via intranet or internet

  • What you see is what you get, no matter what printer use

  • Computing on client site, highly decrease server compute loading

  • Highly integrate form and report system

  • Consistent design rules and data flow in form and report system

  • Visual designers and wizards for drag-and-drop report development

  • Absolute print position on sheet, measured by inch or mm

  • Media, color, orientation, origin, quality page attributes support

  • ISO/DIN and JIS media type support

  • Default printer or dynamic selected printer during run time

  • Printer head offset physical or printable adjustment

  • Report print the data directly came from the form resource which queried or processed by user

  • Printable data include form components, data source, local and globe variable

  • Test preview printing online
