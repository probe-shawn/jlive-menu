データベースTransaction Isolation Levelの設定

　

  • データベース (database) ： データベース選択。

  • Transaction Isolation ： 

    • Transaction none ：コントロールなし。

    • Transaction read uncommitted ：まだ Commit していない資料を読み取れます，データベースに対する制限はもっとも少ないが，多数の人が同時にアクセス可能ですが，データの一致性と完成性に欠けています。

    • Transaction read committed ：

      データは安定するまで，他のアクセスから利用できないが，他のアクセス中のデータの更新を許すことで，例えば，新規、修正、削除など，エラーに導く可能性があります。

    • Transaction repeatable read ：一つのデータアクセスしている際中，他のデータアクセスの修正と削除が許可されない，そうすると，同じデータに同時アクセス機会が少なくなります。

    • Transaction serializable ：一つのデータアクセスしている際中，他のデータアクセスの新規、修正と削除が許可されない，他のアクセスで新規されたデータがあっても，２回クエリー結果が異なることがありません。
