6. フォーム (Form)

内容:

　

新規フォーム(Create new form)

フォームプロパティ (Form properties)

フォーム管理 (Form arrangement)

フレームワークデザイン(Design framework)

　

フォーム画面(Form) は分散式アプリケーションの基本モジュールで，アプリケーションのインタフェースでもあります。一つアプリ流れ(System flow)の中で、機能の異なる各種のフォーム (Form)から構成されています 。マイクロソフト製品の場合，フォームは基本の操作インタフェースですが，Java の場合はパネルと呼ばれます，ここはデザインナーの習慣を考慮し，私たちはパネルと呼びます。

　

6.1 新規フォーム (Create new form)

　

新しいフォームの作成。

【フォーム】の節を選択し，右下の 新規フォーム をクリックします 。 (その後，フォーム設定画面が表示されます)

§   取り消　　　 ( 操作取り消し )

§   新規  　　　　 ( 新規フォーム )

§   削除 　　　　 ( フォーム削除 )

§   保存 　　　　 ( フォーム設定保存 )

§   作成  　　　  ( Java コード作成　 Xml --> Java ) XML から Java ソースコードを作成します。

§   ビュー 　　    ( Java ソースコードのビュー ) 作成されたJavaソースコードビュー。

§   コンパイル 　 ( Javaコンパイル Java --> class ) コンパイルJava プログラム。

§   ビルタ 　　     ( classビルト  Xml --> Java --> class ) XMLからJavaソースコードに，その後，コンパイルclass。

　　

§   Class 名 (Class Name) : フォーム名の登録，フォーム名が空白できません。必ず英文でJava変数の原則に従って登録します，システムはこれによってJava classを作成します ，しかも，アプリケーション中に参照されます。既に存在しているフォームclass名を修正する時に ，システムは２重にならないようにチェックします。

　

§   作者 (Author) : デザインナー名前

　

§   バージョン (Version) : フォームバージョン別

　

§   分類 (Category) : フォーム分類，コントロールパネル上に一時に隠れることが可能

　

§   XML 檔 (XML File) : 一回目にフォームをコンパイルする時に，XMLファイルが作成され，XML中にフォームのプロパティなどが記入されています。

　

§   Java 程式碼 (Java Source) : システムによって作成されるJavaソースコード

　

§   Class パス (Class Path) : Java class保存ディレクトリーの指定と作成

　

§   ファイル記述 (Description) : デザインナーはフォームに対する釈明
