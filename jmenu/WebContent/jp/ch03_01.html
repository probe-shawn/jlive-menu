オブジェクト指向プログラミング (object oriental programming)

オブジェクト (object)

  • ディスプレイとフォーマット (display format)：位置、サイズ、フォント、カラー、隠し、資料表示方式。
  • 内部プロセス (internal process)：initial and destroy、data stream、authorized enable、data logic。
  • データ関連 (data aware)：データコネクション、フェッチ、アップデード、認証。
  • システムとレスポンスイベント (system and response event)：システムフローイベント、ユーザ操作レスポンスイベント。

ＭＶＣ(Model/View/Controller) 構造 (object Model/View/Controller architecture) 

  • Model：システム携帯，潜在的なロジスティックス。
  • View：Model ビュー。
  • Controller：ユーザは Model との連動。

オブジェクトデザイン (object design)

  • オブジェクト作成 (create object )
    • 新規オブジェクト (create object)：新規フォーム (form)、レポート (report)、データソース (data source)、コンポーネント (component)、イベント (event)。
  • View
    • 表示フォーマット (display format)：フォーム、レポートプロパティ設定 (form and report  properties)及びコンポーネント外見プロパティ設定 (format properties)。
  • Model
    • データ関連 (data aware)： コンポーネントデータ連結 (data binding)、データソース SQL コマンド (SQL command)、ダウン・アップロードファイル (download and upload file)。
    • 内部プロセス (iternal process)：データソースフェッチ (fetch)、編集 (update)、新規追加 (insert)、削除 (delete)、SQL error handle。
  • Controller
    • システム及び反応イベント (system and response event)：フォームオープン前後及びクローズイベント (before and after open event) 、コンポーネント起動イベント (component fire event)。

イベントドライブ (event drive)

イベントフローの起動：

  1. ユーザ起動 (user fired)：ユーザはフォームコンポーネントを操作すると，各種類のコンポーネントを起動できます，例えば：キークリック操作或いはデータ更新など。

  2. システム起動 (system fired)：フォームのオープン・クローズ、データソースフェッチ・更新・削除など、 SQL 実行エラーなど。

  3. タイマーコンポーネント起動 (timer component fired)：タイマーコンポーネントは設定された時間と条件の元で起動されます。
