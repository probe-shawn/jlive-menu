Quick Start

　
1. 安裝最新版之J2SDK

　

目前 Java™ 最新的版本為 J2sdk FCS 1.4.1_01，可以至下列網址下載：

http://java.sun.com/j2se/1.4.1/download.html

執行 j2sdk-1_4_1_01-windows-i586.exe (Windows) 或 j2sdk-1_4_1_01-linux-i586-rpm.bin (Linux) 並按照指示安裝 Java™ J2sdk。

　
2. 安裝 jLIVE Builder™
　

執行 Setup.bat (Windows) 或 jSetup.jar (Linux) 並按照指示安裝 jLIVE Builder™。

建議使用預設路徑：

Windows：c:\jLIVE\
Linux： /usr/jlive/

使用者名稱及授權序號：

測試版：serialno.txt內含於安裝光碟 (setup disk) 中。
正式版：serialno.txt內含於授權光碟 (licensed disk) 中。

安裝完成後，正式版授權用戶請複製授權光碟中jLIVE/jalive.jar 貼到 C:\jLIVE\jar\ 中覆蓋原同名檔案 。　

　
3. 開始使用 jLIVE Builder™ 前

  

選擇安裝 third party 軟體：

Apache Tomcat：jLIVE™  middle ware (jslive) container。
隨附於 jLIVE Builder™ 安裝光碟的 Apache Tomcat 版權聲明位於 c:\jLIVE\tomcat\LICENSE。

Apache James  SMTP (選擇性安裝)：只應用於 smtp mail 測試使用。
隨附於 jLIVE Builder™ 安裝光碟的 Apache James 版權聲明位於 c:\jLIVE\james\LICENSE。

mySQL™ 資料庫 (選擇性安裝) ： 只應用於範例資料庫測試使用。
下載網址 -- http://www.mysql.com/downloads/index.html。

mySQL™ Connector/J 驅動程式(選擇性安裝) ： 只應用於範例資料庫連結測試使用。
下載網址 -- http://www.mysql.com/downloads/api-jdbc-stable.html。

IDAutomationLinear Java Barcode Package (選擇性安裝)：Barcode 測試及評估使用 (測試版)。
下載網址 -- http://www.idautomation.com/java/。

i-net JDBC  驅動程式(選擇性安裝) ：MSSQL 資料庫驅動程式 (測試版)。
下載網址-- http://www.inetsoftware.de/English/produkte/JDBC2/Default.htm


安裝 mySQL™ 資料庫

  1. 下載 mysql-3.23.53-win.zip (安裝 zip 檔) 。
  2. 解壓 mysql-3.23.53-win.zip 並安裝 mySQL™ 資料庫於目錄 c:\mysql\。
  3. 複製 c:\jLIVE\mysql\data\northwind 目錄及其下的所有檔案至 c:\mysql\data\目錄中。
  4. 複製 c:\jLIVE\mysql\data\mysql 目錄及其下的所有檔案至 c:\mysql\data\目錄中並覆蓋原有檔案。


安置 Connector/J mySQL JDBC 驅動程式

  1. 下載 mysql-connector-java-2.0.14.zip (驅動程式 zip 檔) 。
  2. 解壓 mysql-connector-java-2.0.14.zip。
  3. 複製 mysql-connector-java-2.0.14-bin.jar 至 c:\jLIVE\tomcat\lib\目錄中。

安置 IDAutomationLinear Java Barcode Package 

  1. 下載 JavaBarCodeDemo.zip (package zip 檔) 。
  2. 解壓 JavaBarCodeDemo.zip。
  3. 複製 LinearBarCode.jar  至 c:\jLIVE\jar\目錄中。

安置 i-net MSSQL JDBC 驅動程式

  1. 下載 Sprinta2000_Trial.zip (驅動程式 zip 檔) 。
  2. 解壓 Sprinta2000_Trial.zip。
  3. 複製 Sprinta2000.jar 至 c:\jLIVE\tomcat\lib\目錄中。

　

4. 設定環境變數 (JAVA_HOME 和 CATALINA_HOME)
　

Windows NT/2000/XP 設定方式

  1. 【控制台】->【系統】->【進階】-> 環境變數。按下新增鈕新增系統變數。
  2. 新增系統變數
    變數名稱：JAVA_HOME         變數值：c:\j2sdk1.4.1_01
    變數名稱：CATALINA_HOME  變數值：c:\jLIVE\tomcat

Windows 95/98/me 設定方式

  1. 編輯 c:\autoexec.bat。
  2. 加入 SET CATALINA_HOME=C:\jLIVE\tomcat 和 SET JAVA_HOME=C:\j2sdk1.4.1_01。
  3. 重新開機。

　

5. 啟動範例資料庫 mySQL™
　
  1. 執行 c:\mysql\bin\winmysqladmin.exe。
  2. 第一次啟動資料庫會要求使用者輸入帳號及密碼，請輸入大寫帳號： PROBE ，密碼：PROBE。

　

6. 啟動 Tomcat 服務 (jLIVE middle ware)
　

執行 c:\jLIVE\tomcat\startup.bat.

　
7. 執行範例程式
　

在jLIVE Builder™ 環境中測試執行

  1.   啟動jLIVE Builder™ ：執行安裝目錄下的 exeJdlive.bat (c:\jLIVE\exejdlive.bat)。
  2. 展開NorthWind_sample 應用程式：在控制面板中點選 NorthWind_sample 的節點，展開後有【資料庫】【視窗】【執行程式】三個節點。
  3. 編輯主視窗：請點選【視窗】節點，展開所有視窗，請標示第一個視窗節點( MAIN_MENU )，按右鍵選擇編輯選單進入設計畫面。
  4. 執行主視窗：進入後在上方功能選單中的【視窗畫面】【執行】，開始執行範例程式。

直接以瀏覽器 (browser) 執行

  1. 滑鼠右鍵雙擊 C:\jLIVE\programs\jNorth\jNorth.html。

　
Trouble Shooting
　
  • 請確定已經正確安裝 j2sdk-1_4_1_01。
  • Window 95/98/me 的使用者，安裝完成後必須重新啟動。
  • Tomcat
    • 請檢查系統變數是否設定正確。( JAVA_HOME和CATALINA_HOME )
    • 最常發生的問題，是 Tomcat HTTP 的預設 Port 8080，已經被其他的 Web server 伺服器所使用，例如： Oracle  9i 的 application server。要改變預設的 Port ，請開啟：
      $CATALINA_HOME/conf/server.xml
      •••搜尋 '8080' ，更改為尚未被使用的 Port。建議最好大於 1024，詳細情形請參照各作業系統說明。
      重新啟動 Tomcat 即可正常作業。必須注意當改變 Port 8080 後，程式在呼叫 Tomcat 服務的 URL 也要一併修改。例如：改 Port 為 1977，呼叫 Tomcat 服務的 URL 為 http://localhost:1977/ 。
    • 在執行 Windows 9X/ME 的batch file 啟動時出現 "out of environment space" 的錯誤。 
      以滑鼠右擊 STARTUP.BAT 和 SHUTDOWN.BAT 的圖示，選擇內容選單項目的【記憶體】設定頁， 將傳統記憶體的起始環境設定成 4096。
      確定或套用後，Windows 將會建立捷徑，可以此捷徑啟動或關閉 Tomcat 。
    • localhost 不存在 ('localhost' machine isn't found)，這是因為使用 Proxy 的緣故。如果是在此情況下，必須要修改瀏覽器的 Proxy 設定，以避開 Proxy 主機。 
      使用 Netscape，請編輯 Edit/preferences -> Advanced/proxies。使用 Internet Explorer，請編輯 Tools -> Internet Options -> Connections -> LAN Settings。
