Applet

System will automatically generate a HTML file which embed a Java applet. User can use the browser to launch the HTML file then applet will be executed. All the forms and reports just install in Web site, that will be downloaded and cached when client reference it. 
JarRun

System will compress all the forms and reports to a executable Java jar file. The executable Java jar file is just like any executable programs in Windows system. 
Applet
Applet run :  Generate a HTML file which embedded a Java applet.
Save :  Save the modified properties.
Security Control :  The default jLIVE security control module.
App Name :  The application name for security control module.
HTML Name :  HTML file name. Developer can modified this html file except the applet tag content.
Applet Name :  Java Applet which embedded in HTML file.
Main Panel :  The application main panel, which will be plugged in browser page, that act as the application home page. 
JRE Plugin :  The URL for download and installation Java Runtime Environment. Default is come from Sun Web.
Custom Jar File :  Third party Java class jar file that will be manipulated in application and must be delivered to client.
Description :  Description.
JarRun
Jar Run :  Generate a executable Java Jar file.
Save :  Save the modified properties.
Security Control :  The default jLIVE security control module.
App Name :  The Executable Java jar file name.
Main class Name :  jLIVE automatically generated Java main()  class name.
Application Title :  The application main window title.
Resizable Frame :  The main window can be resized.
Initial State :  The initial state of main window. 
Main Panel :  The content panel of the application main window.
Custom Jar File :  Third party Java class jar file that will be manipulated in application and must be delivered to client.
