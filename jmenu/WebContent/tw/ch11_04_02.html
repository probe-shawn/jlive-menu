出現訊息方塊 (show message box)

　

事件執行狀態與訊息 (event runtime status and message)

　

訊息盒事件 (message box event) ，提供五種不同的訊息類別 (message type) 圖示 ： 錯誤圖示、訊息圖示、警告圖示、問題圖示和空白無圖示。 選擇類別 (option type) 提供四種按鈕組合，三種事件流程，處理使用者(user) 的回應選擇。使用者回應的事件流程 (YesOk  Event、No Event、Cancel Event)，處理完畢後，系統會接續處理後續事件 (next event)。

  • 標題 (caption)： 訊息方塊的標題。

  • 訊息 (message)： 訊息內容。使用運算式。多行 (multi-line) 訊息以 '\n' 為換行 (linefeed) 字元。

  • 訊息類別 (message Type)： 設定訊息類別，各有不同的圖示顯示在訊息方塊的左上角。

    • 錯誤 (error)： 錯誤圖示。

    • 情報 (information)： 訊息圖示。

    • 警告 (warning)： 警告圖示。

    • 問題 (question)： 問題圖示。

    • Plain ： 空白無圖示。

  • 選擇類別 (option type)： 訊息方塊出現之後，使用者可以選擇的作業方式 ，以按鈕(button)呈現。

    • 確定 (ok)： 顯示【確定】(ok) 按鈕。

    • 是＿否 (yes_no)： 顯示【是】(yes) 與【否】(no) 按鈕。

    • 是＿否＿取消 (yes_no_cancel)： 顯示【是】(yes) 、【否】(no) 、【取消】(cancel) 按鈕。

    • 確定＿取消 (ok_cancel)： 顯示【確定】(ok) 與【取消】(cancel)  按鈕。

  • 是/確定＿事件 (yesok  event)： 使用者(user) 按下【是】(yes) 或者【確定】(ok) 按鈕之後所執行的事件。

  • 否＿事件 (no event)： 使用者(user) 按下【否】(no) 按鈕之後所執行的事件。

  • 取消＿事件 (cancel event)： 使用者(user) 按下【取消】(cancel)  按鈕之後所執行的事件。

事件執行狀態與訊息 (event runtime status and message)

　

EvStatus("事件名稱") ：取得事件執行狀態。
　

回傳值 狀態
空字串 事件未被引發。
-1 事件啟動後，依條件判斷後執行否則事件。
1 事件啟動後，執行成功。
0 事件啟動後，執行過程有誤。

　

EvStatus("事件名稱.error") ：取得事件執行錯誤代碼。

　

回傳值 說明
空字串 或 0 沒有錯誤訊息或無法取得錯誤訊息。

　

EvStatus("事件名稱.INFO_STRING") ：取得事件執行 INFO_STRING 的訊息。

　

INFO_STRING 回傳值 說明
option -1或 0 或 1 -1 ：使用者按選 【取消】。
0 ：使用者按選【否】。
1 ：使用者按選【確定】或【是】。
