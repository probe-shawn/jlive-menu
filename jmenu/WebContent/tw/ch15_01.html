jLIVE™ middle ware 啟動與結束 (jLIVE™ middle ware startup and shutdown)

　

jLIVE™ middle ware 的啟動 (startup)、結束 (shutdown) 與存在期 (life cycle) 由應用伺服器 (application server) 所負責。相關的啟動方式與結束，請參照各應用伺服器中對啟動 Servlet 的說明。例如：在 Tomcat 安裝完jLIVE™ middle ware 後 ，執行 Tomcat 中的 startup.exe 啟動 Tomcat 服務，在第一次使用者端請求服務時， Tomcat 會啟動 jLIVE™ middle ware 初始化程序，有些應用伺服器可以在應用伺服器啟動後，立即啟動 jLIVE™ middle ware 初始化程序。結束 Tomcat 的 jLIVE™ middle ware 服務，則執行 shutdown.exe ，jLIVE™ middle ware 會執行結束程序。
　

jLIVE™ middle ware 啟動初始化程序

  • 執行 Java Servlet 組態初始化 (Servlet initialize)。

  • 查詢系統環境 (Java 版本、OS 版本)。

  • 是否支援加密 (encryption supported)。

  • 使用授權檢驗 (authorized license)。

  • 讀取及建立 jLIVE™ middle ware 屬性設定檔 (properties)。

  • 建立語系管理 (language management)。

  • 建立應用程式管理 (application management)。

  • 建立使用者管理 (user management)。

  • 建立使用者服務作業管理 (user service handle management)。

  • 建立 Time out 管理 (time out management)。

jLIVE™ middle ware 結束程序

  • 執行 Time out 管理結束程序 (close time out management)。

  • 執行 使用者服務作業管理結束程序 (close user service handle management)

  • 執行 使用者管理結束程序 (close user management)

  • 執行 應用程式管理結束程序 (close application management)

  • 釋放連結資源  (close connection)。

  • 記錄作業狀態 (log status)。

  • 執行 Java Servlet 結束程序 (destroy)。

jLIVE™ middle ware Timeout 管理 (jLIVE™ middle ware Time out management)

　

jLIVE™ middle ware 在使用者端發出連線服務請求 (connection request) ，並經檢驗為合法使用者後，隨即在系統中建立一個使用者服務作業模組 (user service handle)，使用者端接續的所有服務請求，皆由此專屬使用者服務作業模組提供服務。例如：使用者資訊、加密 Session key 、使用者資料來源結果集 Cache 機制、交易 (Transaction)、使用者存取 log •••等。

使用者端發出服務請求時，jLIVE™ middle ware 會先尋找使用者服務作業模組 (user service handle)，並將服務請求轉交其提供服務，如果jLIVE™ middle ware 無法找到專屬使用者服務作業模組，jLIVE™ middle ware 會回應使用者操作逾時訊息，並停止提供服務。一般正常狀態使用者端不會產生操作逾時，通常是使用者不正常斷線或網路斷線後，沒有再經連線服務請求 (connection request) 的程序，隨即傳送要求作業服務請求 (request service)。

jLIVE™ middle ware 在啟動後，會建立 Time out 管理 (time out management)。Time out 管理的 Thread 每隔 1分鐘會啟動 Timeout 檢查，在下列狀況下：

  • 使用者 5 分鐘內未要求服務且未有心跳連線 (heart beat connection)。

  • 後端資料庫服務超過 15 分鐘。

  • jLIVE™ Builder 視窗設計平台，超過 8 小時未要求服務。

會移除使用者服務作業模組 (user service handle)，使 jLIVE™ middle ware 在系統資源使用上有最佳狀態。所以使用者如果不正常關機或網路斷線，最慢在 5 分鐘內， jLIVE™ middle ware  會清除其所用的所有資源，系統管理師亦可經由  jLIVE™ middle ware 管理介面，在任何時刻移除任何使用者服務作業模組。

jLIVE Builder™ 的使用者端應用程式，在使用者操作應用程式時，如果使用者端未在 3 分鐘內，發出任何服務請求時，使用者端系統會自動產生一心跳連線請求 (heart beat connection) 至 jLIVE™ middle ware ，通知jLIVE™ middle ware 使用者端仍在作業，尚需  jLIVE™ middle ware 提供服務， jLIVE™ middle ware 在 Timeout 管理時不會移除此使用者。

jLIVE™ middle ware  屬於資料庫存取的服務會連結後端資料庫系統，轉請其服務並等待回應，如果回應等待時間超過 15 分鐘，可能是後端資料庫系統有不可知的錯誤產生 (一般錯誤會以 SQL Exception 回應)， jLIVE™ middle ware  會移除此使用者服務作業模組，並回應使用者端操作逾時訊息。

jLIVE™ Builder 視窗設計平台，並不會產生心跳連線請求 (heart beat connection) ，系統為避免 jLIVE™ middle ware Timeout 管理造成視窗設計平台操作逾時，設計畫面無法正常存檔，所以 8 小時 才會 Timeout。設計師如果在設計時有不正常的關機或網路斷線，可以使用 jLIVE™ middle ware  管理介面，移除應用程式名稱為 jdlive 的使用者。

jLIVE™ middle ware 服務請求與回應 (jLIVE™ middle ware service request and respond)

　

jLIVE™ middle ware 初始化後，隨即提供使用者的服務請求與回應。使用者端發出的第一個請求，是連線服務請求 (service connection request) ，並傳入以下資訊：

  • 使用者 IP。

  • 應用程式名稱。

  • 登入名稱、密碼。

  • 資料庫資訊。

  • 加密協商資訊。

jLIVE™ middle ware 回應程序如下：

  1. 根據 jLIVE™ middle ware 所記錄應用程式名稱資訊，查驗是否須作登入者安全控管。

  2. 如果使用 jLIVE 安全控管 (jLIVE™ security control)

    • 查詢使用者是否合法。

    • 擷取授權資訊 (authorized items)。

    • 查詢登入名稱、密碼的資料庫登入名稱、密碼。

  3. 非法使用者，回傳使用者名稱、密碼不正確訊息，返回。

  4. 建立使用者服務作業模組 (user service handle)。

  5. 回傳加密協商資訊、授權資訊、連線服務請求成功訊息，返回。

使用者端在發出第一個請求後，隨即可以要求 jLIVE™ middle ware 提供應用程式所需的服務。jLIVE™ middle ware 所提供的服務分為系統資源服務及資料庫伺服器服務。服務項目代碼請參照 jLIVE™ middle ware 服務代碼。

　

系統資源服務

  • 檔案上下載 (file download and upload)。

  • 應用伺服器所在的區域網路，檔案系統服務 (file system service)。

  • Cookie 服務 (cookie service)。

  • SMTP 服務 (smtp service)。

  • 應用伺服器環境資訊 (application server environment)。

  • 程式設計期服務

    • 測試連結 (test connection)。

    • 上下載應用程式 XML 資源 (upload and download application XML)。

    • 檢驗程式管理員 (application administrator)。

    • 授權人數 (authorized concurrent user)。

    • 目前線上人數 (user count)。

資料庫伺服器服務

  • 開啟資料庫連結 (open  connection)

  • 關閉資料庫連結 (close connection)

  • 開啟資料來源 (open data source)

  • 關閉資料來源 (close data source)

  • 整批開啟資料來源 (batch open data source)

  • 整批關閉資料來源 (batch close data source)

  • 取得資料來源 Meta data (get data source meta data)

  • 資料來源，取得起迄資料記錄的資料值 (get record data)

  • 資料來源，新增資料記錄 (insert record data)

  • 資料來源，更新資料記錄(update record data)

  • 資料來源，刪除資料記錄 (delete record data)

  • 查詢資料來源 (query data source)

  • 執行 SQL 命令集 (execute SQL command)

  • 執行資料庫系統的預儲程序 (execute stored procedure)

  • 執行資料庫系統Auto Commit 的設定與取消 (auto commit on or off)

  • 執行資料庫系統 Commit (commit)

  • 執行資料庫系統 Roll back (roll back)

  • 執行資料庫系統 Isolation level (isolation level)

  • 程式設計期服務

    • 資料庫系統 連結測試 (data base connection test)

    • 資料庫系統 Meta data (data base meta data)

    • 資料庫系統資料表資訊 (data base table information)

    • 資料庫系統資料表欄位資訊 (data base table column information)

    • 資料庫系統資料表索引資訊 (data base table index information)

    • 資料庫系統預儲程序資訊 (data base stored procedure information)

    • 資料庫系統預儲程序名稱 (data base stored procedure name)

    • 資料庫系統預儲程序欄位資訊 (data base stored procedure column information)
