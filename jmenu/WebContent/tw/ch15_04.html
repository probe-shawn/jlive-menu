15.4 jLIVE™ middle ware 安裝 (jLIVE™ middle ware installation)

安裝於 BEA WebLogic™ (BEA WebLogic™ setup jLIVE™ middle ware)
安裝於 WebSphere™ (WebSphere™  setup  jLIVE™ middle ware)
安裝於 Interstage™ (Interstage™ setup jLIVE™ middle ware)

 jLIVE™ middle ware 是由 pure Java™ 所撰寫的 Servlet 模組 -- jslive.jar ，以 JAR 檔的型式掛載於應用伺服器。jslive.jar 包含 Servlet 、系統資源服務、資料庫伺服器服務 及相關授權的 Java™ class。完整的中間層 (middle ware) 尚需使用 SMTP 及 存取資料庫所需的 JDBC 驅動程式，程式列示如下： 

  • jLIVE™ middle ware

    • jslive.jar
  • SMTP
    • mail.jar
    • mailapi.jar
    • imap.jar
    • pop3.jar
    • smtp.jar
  • JDBC
    • MSSQL
      • mssqlserver.jar
      • msbase.jar
      • msutil.jar
    • ORACLE
      • ojdbc14.jar
    • mySQL
      • mysql-connector-java-3.0.14-production-bin.jar

其中 jslive.jar 為必須的 JAR 檔；程式中使用 Send Mail 事件時，必需加入 SMTP 以下的 JAR 檔； JDBC 則視應用程式所使用的資料庫系統而定，以上所列的 JDBC 為各資料庫系統的官方驅動程式，使用其他廠商的 JDBC 可以相同方式安裝或參照其說明文件。以上所列示檔案安置於 jLIVE/tomcat/common/lib/ 目錄下。

安裝 jLIVE™ middle ware 於應用伺服器時，請參照各應用伺服器關於安裝 Servlet service 的說明文件， jLIVE™ middle ware 的 Servlet class 名稱為 com.probe.jslive.ProbeServlet，按文件說明將 jslive.jar 安裝成應用伺服器的一個 Servlet instance 即可。SMTP 相關 JAR 檔則置放於應用伺服器共同模組的 LIB 目錄中。JDBC 模組則參照各應用伺服器安裝 JDBC 文件說明，安裝 JDBC ，或直接將其置放於應用伺服器共同模組的 LIB 目錄，或置放於應用伺服器 Web 模組 WEB-INF/lib 下。所有符合 J2EE 規格的應用伺服器，皆有詳細的 Servlet 安裝說明文件，設計師可以參考以下所附關於 WebLogic 、WebSphere 及 Interstage 的安裝說明。

 jLIVE™ middle ware 在 Tomcat™ 的安裝，只要將上述所有 JAR 檔置放於 tomcat/common/lib/ 目錄下即可，相關的其他 Tomcat™ 設定請參照  Tomcat™ 說明文件及 jLIVE Builder™ 安裝與啟動。

安裝於 BEA WebLogic™ (BEA WebLogic™ setup jLIVE™ middle ware)

　

步驟一 (Step 1)：安裝 BEA WebLogic platform。

  1. 測試版的網址：http://commerce.bea.com/downloads/products.jsp。

　

步驟二 (Step 2)：建立一個新的 WebLogic Domain。

  1. 選擇執行選單【開始】【程式集】【BEA WebLogic Platform7.0】【Domain Configuration Wizard】進入精靈視窗。

  2. Choose Domain Type And Name：選擇建立 WLS Domain 的 template，並命名為  jlive_domain。

  3. Choose Sever Type：選擇 Single Server (Standalone Server)

  4. Choose Domain Location：以預設路徑即可。

  5. Configure Standalone/Administrator Server：
    Server Name：jlive_server
    Server Listen Address：
    Server Listen Port：80 (或 8080)
    Server  SSL Listen Port：8081 

  6. Create Administrative User：
    User Name：probe_admin
    Password：probe_admin
    Verify Password：probe_admin

  7. Install Server as Windows Service：No

  8. Create Start Menu Entry For Server：Yes

  9. Configiguration Summary：按 Create 鍵，建立 Domain。

步驟三 (Step 3)：修改 web.xml。

  1. 建立 Domain 後，在 bea\user_projects\ 目錄下會新增一 jlive_domain 目錄。
  2. 以 Notepad 開啟 bea\user_projects\jlive_domain\applications\DefaultWebApp\WEB-INF\web.xml。
  3. 將 web.xml 修改為以下：
    　
    <?xml version="1.0" ?>
    <!DOCTYPE web-app PUBLIC "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN" "http://java.sun.com/dtd/web-app_2_3.dtd">

    <web-app>
        <servlet>
            <servlet-name>
                ProbeServlet
            </servlet-name>
            <servlet-class>
                com.probe.jslive.ProbeServlet
            </servlet-class>
        </servlet>
        <servlet-mapping>
            <servlet-name>
                ProbeServlet
            </servlet-name>
            <url-pattern>
                /servlet/*
            </url-pattern>
        </servlet-mapping>
        <servlet-mapping>
            <servlet-name>
                ProbeServlet
            </servlet-name>
            <url-pattern>
                /servlet/com.probe.jslive.ProbeServlet/*
            </url-pattern>
        </servlet-mapping>
    </web-app>

步驟四(Step 4)：建立 jLIVE Builder middle ware。

  1. 在上述 WEB-INF 目錄下新增兩個目錄 \lib\ 及\classes\。
  2. 將 下列 jar file 複製到 \lib\ 目錄中
    • jslive.jar (jLIVE middle ware)
    • mailapi.jar (SMTP mail api)
    • mysql-connector-java-3.0.14-bin.jar(MySQL driver) 
    • Sprinta2000.jar (MSSQL Driver)
  3. 將以 class 存在的 library 檔複製到 \classes\ 目錄中，例如 oracle driver。

步驟五(Step 5)：建立 httpd 的應用程式模組。

  1. 將由 jLIVE Builder™ 所產生的應用程式的目錄下，啟動程式的 *.html 及class 目錄下所有檔案，複製到 bea\user_projects\jlive_domain\applications\DefaultWebApp\下 。
  2. 以 jLIVE Builder Sample 為例：將 jLIVE\programs\jNorth\jNorth.html 及jLIVE\prograqms\jNorth\class\ 目錄及以下所有檔案複製到 bea\user_projects\jlive_domain\applications\DefaultWebApp\下 。

步驟六(Step 6)： Java™ 版本 ，jLIVE Builder™ middle ware 需要 j2sdk 1.4.0 以上版本。

  1. 修改 bea\weblogic700\server\bin\startWLS.cmd，將 set JAVA_HOME=c:\bea\jdk131_03 改為 set JAVA_HOME=c:\j2sdk1.4.2(安裝的 1.4.0 版本以上的 Java 目錄) 。
  2. 將修改完成的檔案另存成 bea\weblogic700\server\bin\startWLS_jlive.cmd。
  3. 修改 bea\user_projects\jlive_domain\startWebLogic.cmd，將檔案最後段的 call "c:\bea\weblogic700\server\bin\startWLS.cmd" 改為 "c:\bea\weblogic700\server\bin\startWLS_jlive.cmd"。

步驟七(Step 7)： Security Policy 。

  1. 修改 Security Policy file 

步驟八(Step 8)： 啟動 WebLogic Server 。

  1. 選擇執行選單【開始】【程式集】【BEA WebLogic Platform7.0】【User Projects】【jlive_domain】【Start Server】
  2. 在啟動的 DOS BOX 中輸入 username ：probe_admin 及 password：probe_admin。
  3. 在 DOS BOX 中，WebLogic 會出現 <Server started in RUNNING mode> 表示 WebLogic 以啟動。

步驟九(Step 9)： 測試 jLIVE Builder middle 及 應用程式 。

  1. 開啟 Browser， 輸入URL ：http//localhost (或 :8080)/servlet/com.probe.jslive.ProbeServlet/，Browser 將顯示 Probe Application Server Administrator 的登入畫面，表示 jLIVE middle ware 以正常啟動。
  2. 開啟 Browser， 輸入URL ：http//localhost (或 :8080)/jNorth.html ，Browser 將啟動 jNorth 應用程式。

安裝於 WebSphere™ (WebSphere™  setup  jLIVE™ middle ware)

　

安裝前 (Before setup Websphere)：請先按 [jslive 授權光碟]\jslive\ Middleware 快速安裝.doc ，安裝 middle ware keypro 所需程式模組，否則 jLIVE middle ware 將會自行調整為 light version for development only，屆時只能服務一個線上使用者。

　

　

步驟一 (Step 1)：安裝 IBM WebSphere™ Application Server version 4.0 Advanced Edition Single Server 

  1. 測試版的網址：http://www7b.boulder.ibm.com/wsdd/downloads/evaluation.html#WAS。

  2. IBM HTTP server 請一併安裝，應用程式模組 (Html and java classes)  ，將由 IBM HTTP server 啟動。

　

步驟二 (Step 2)：建立 J2EE Web Application Module -- jslive.war 。

  1. 選擇執行選單【開始】【程式集】【IBM WebSphere】【應用程式伺服器 V40 AES】【應用程式組合工具】。

  2. 在【一般】標籤頁中選擇 Web 模組，按確定鍵進入應用程式組譯器視窗。

  3. 在右下方的【新建】標籤頁中填入
    顯示名稱：ProbeServlet
    可分送：核取 (checked)
    類別路徑：jslive.jar
    按套用鍵後，左邊樹狀根節點會更名為 ProbeServlet。

  4. 按選標示 (mark) Web 元件節點後，按滑鼠右鍵選擇【新建】選單。

  5. 在新建 Web 元件視窗的【一般】標籤頁中填入
    元件名稱：ProbeServlet
    顯示名稱：ProbeServlet
    元件類型：核選 Servlet
    類別名稱：com.probe.jslive.ProbeServlet
    按確定鍵後離開新建 Web 元件視窗。

  6. 按選標示 (mark) Servlet 對映 節點後，按滑鼠右鍵選擇【新建】選單。

  7. 在新建 Servlet對映視窗的【一般】標籤頁中填入
    URL 型樣：/*
    Servlet：選取 ProbeServlet
    按套用鍵，套用此設定後，再次填入
    URL 型樣：/com.probe.jslive.ProbeServlet/*
    Servlet：選取 ProbeServlet
    按確定鍵，套用後，離開新建 Servlet對映視窗。

  8. 展開 檔案 節點，按選標示 (mark) JAR 檔 節點後，按滑鼠右鍵選擇【新增檔案】選單。

  9. 在新增檔案視窗中，按瀏覽選取目錄 [jslive 授權光碟]\jslive\jre141\lib
    根目錄或保存檔：[jslive 授權光碟]\jslive\jre141\lib
    在右視窗中會出現所有 jar 檔。
    jslive.jar
    imap.jar
    mail.jar
    mailapi.jar
    mysql-connector-java-3.0.14-bin.jar
    pop3.jar
    smtp.jar
    Sprinta2000.jar
    按新增鍵，將所有 jar 檔加入於選取的檔案列示中。
    按確定鍵，離開新增檔案視窗。

  10. 展開 檔案 節點，按選標示 (mark) 類別檔 節點後，按滑鼠右鍵選擇【新增檔案】選單。

  11. 在新增檔案視窗中，按瀏覽選取目錄 [jslive 授權光碟]\jslive\jre141\classes
    根目錄或保存檔：[jslive 授權光碟]\jslive\jre141\classes
    在右視窗中會出現所有檔案。
    ServletCallTest.class
    按新增鍵，將所有檔案加入於選取的檔案列示中。
    按確定鍵，離開新增檔案視窗。

  12. 按【檔案】【另存新檔】選單，將檔案存於 c:\temp，檔名為 jslive.war。 

  13. 離開應用程式組合工具。

步驟三 (Step 3)：在 WebSphere 中建立 jLIVE™ middle ware Web 應用程式。

  1. 選擇執行選單【開始】【程式集】【IBM WebSphere】【應用程式伺服器 V40 AES】【啟動應用程式伺服器】。
  2. 等待 DOS BOX 中出現 “暫停中；請按 Enter 鍵，繼續作業”訊息。
  3. 選擇執行選單【開始】【程式集】【IBM WebSphere】【應用程式伺服器 V40 AES】【管理主控台】。
  4. 填入安裝時登入 ID ，登入應用程式伺服器管理主控台。
  5. 展開左方 [節點] 節點的樹狀結構至出現 [企業應用程式] 節點，按選該節點，右方會出現企業應用程式的管理視窗。
  6. 按【安裝】鍵後，視窗出現應用程式安裝精靈。填入以下屬性
    路徑：c:\temp\jslive.war (由步驟二所製作的 Web 模組)
    應用程式名稱：ProbeServlet
    環境定義根目錄：/servlet
  7. 按下一步鍵。
  8. 在指定Web模組的虛擬主電腦和經過前置編譯的 JSP 選項。按下一步。
  9. 按完成鍵。
  10. 在隨後的視窗上
    若出現：必須儲存配置，則按該選項，並於隨後出現的視窗中按確定儲存配置。
    若出現：必須重新產生外掛程式配置，則按該選項，並於隨後出現的視窗中按產生鍵。
  11. 按結束離開應用程式伺服器管理主控台。
  12. 完成後請檢視【WebSphere 安裝路徑】【AppServer】【installedApps】是否已建立以下目錄及檔案
    ProbeServlet.ear
    |---META-INF
    |---jslive.war
         |---META-INF
         |---WEB-INF
              |---classes
              |---lib
    其中 classes 目錄下含在組合應用程式時，所選入的檔案。
    其中 lib 目錄下含在組合應用程式時，所選入的檔案。
    在下一步驟將根據 Java 版本之不同，將所需的相關檔案複製至此兩目錄中。

步驟四(Step 4)：Java 版本檢查與調整 (Java version check and patch)。

由於 WebSphere Java VM 是由 IBM 所發展的版本，與 Sun Java 的版本不盡相同，jLIVE middle ware 是 Sun Java 1.41 版本，如果您所使用的 WebSphere Java 版本是 1.41 版，請直接進行步驟五。

  1. 檢查 Java 版本。
    在 DOS BOX 中，change directory (cd) 至【WebSphere 安裝路徑】【AppServer】【java】【bin】，輸入 java -version 。如果 版本小於 1.4 請進行以下步驟。
  2. 加入 xml jar 檔。
    將 [jslive 授權光碟]\jslive\jre141\lib_131\xml 的所有檔案複製至 【WebSphere 安裝路徑】【AppServer】【installedApps】【ProbeServlet.ear】【jslive.war】【WEB-INF】【lib】目錄下。
  3. 加入 JCE jar 檔。
    將 [jslive 授權光碟]\jslive\jre141\lib_131\jre13\lib 目錄下的 jce.jar 及jsse.jar 複製至 【WebSphere 安裝路徑】【AppServer】【java】【jre】【lib】目錄下。
    將 [jslive 授權光碟]\jslive\jre141\lib_131\jre13\lib\ext 目錄下的 sunjce_provider.jar 複製至 【WebSphere 安裝路徑】【AppServer】【java】【jre】【lib】【ext】目錄下。
    將 [jslive 授權光碟]\jslive\jre141\lib_131\jre13\lib\security 目錄下的 local_policy.jar 及 US_export_policy.jar 複製至 【WebSphere 安裝路徑】【AppServer】【java】【jre】【lib】【security】目錄下。

　

步驟五(Step 5)：重新啟動 WebSphere 及 IBM HTTP SERVER。

  1. 選擇執行選單【開始】【程式集】【IBM HTTP server】【啟動 HTTP Server】。
  2. 在【WebSphere 安裝路徑】【AppServer】【bin】下，執行 stopServer.bat。
  3. 在【WebSphere 安裝路徑】【AppServer】【bin】下，執行 startServer.bat。

步驟六(Step 6)：測試 jLIVE™ middle ware。

  1. 等待 DOS BOX 中出現 “暫停中；請按 Enter 鍵，繼續作業”訊息。
  2. 開啟 IE ，在 URL 中輸入
    HTTP://localhost
    IE 將會出現 IBM HTTP Server 歡迎使用的畫面，表示 HTTP Server 服務正常 。
  3. 在 URL 中輸入
    HTTP://localhost/servlet/com.probe.jslive.ProbeServlet/
    IE 將會出現 jLIVE™ middle ware 的管理模組，輸入 ID Password 後，按選Server Configuration 。
  4. 檢視 Authorized ID 及 Authorized concurrent users 是否為所購買的授權數。
  5. 如果出現任何無法啟動或非預期的狀況，請將【WebSphere 安裝路徑】【AppServer】【logs】目錄下的記錄檔傳至 service@probe.com.tw。

　

關於應用程式 deployment 

  1. 由 jLIVE Builder™ 所產生的執行模組包括 *.html 及以下 class 的所有檔案，可安置於 HTTP Server。
    將其複製於 [IBM HTTP Server 安裝路徑]\htdocs\下。

  2. IBM HTTP server 及WebSphere 預設是運作於 port 80，所以請檢查應用程式開發時的 middle ware HTTP 是否已改為 80 port。

關於 JDBC driver 

  1. 可使用 WebSphere 已安裝的 JDBC Driver。

  2. 以 WebSphere 安裝 JDBC Driver。

  3. 若只使用於 jLIVE™ 系統，可以直接將 JDBC jar 檔，複製於[WebSphere 安裝路徑]\AppServer\installedApps\ProbeServlet.ear\jslive.war\WEB-INF\lib 中，若是 class 檔則複製至[WebSphere 安裝路徑]\AppServer\installedApps\ProbeServlet.ear\jslive.war\WEB-INF\classes 中。
    例如：
    安置 middle ware 中的 oracle driver ：將 [jslive 授權光碟]\jslive\jre141\oracle_driver 目錄下的所有檔案複製至 【WebSphere 安裝路徑】【AppServer】【installedApps】【ProbeServlet.ear】【jslive.war】【WEB-INF】【classes】目錄下。

安裝於 Interstage™ (Interstage™ setup jLIVE™ middle ware)

　

安裝前 (Before setup Interstage)：請先按 [jslive 授權光碟]\jslive\ Middleware 快速安裝.doc ，安裝 middle ware keypro 所需程式模組，否則 jLIVE middle ware 將會自行調整為 light version for development only，屆時只能服務一個線上使用者。

　

　

步驟一 (Step 1)：安裝 Fujitsu Interstage ™。  

  1. 以 Easy installation 或 Custom installation 安裝後所預設的 Web server 並不相同。

  2. 以下步驟係以 Easy installation 安裝為示範，Custom installation 若使用不同的 Web server 請查閱 Interstage 文件，以相對應所需設定值輸入。

　

步驟二 (Step 2)： 設定 Web Servlet Gateway。

  1. Setting httpd.conf

    1. [Windows] ：
      使用 Interstage HTTP Server
      編輯 [c:\INTERSTAGE]\F3FMihs\conf\httpd.conf
      在 httpd.conf 最後加入以下兩行字串

      LoadModule jsvlt_module C:/INTERSTAGE/F3FMjs2/gateway/jsgw_apapi_is.dll
      include C:/INTERSTAGE/F3FMjs2/conf/jsgw_apapi.conf

      [      Solaris] ：
      使用 Interstage HTTP Server
      編輯 /opt/FJSVihs/conf/httpd.conf
      在 httpd.conf 最後加入以下兩行字串

      LoadModule jsvlt_module /opt/FJSVjs2/gateway/jsgw_apapi.so
      include /opt/FJSVjs2/conf/jsgw_apapi.conf 


    2. 使用 InfoProvider Pro
      參閱 j2ee.pdf -- Chapter 9 (9-6)。

    3. 使用 Internet Information Server
      參閱 j2ee.pdf -- Chapter 9 (9-8)。

    4. 使用 iPlanet Web Server
      參閱 j2ee.pdf -- Chapter 9 (9-8)。

  2. 如果 Servlet Container 與 Web Server 不在同一主機，其設定請參閱 j2ee.pdf -- Chapter 9 (9-10)。

　

步驟三 (Step 3)：設定 Servlet Gateway Definition。

  1. Setting jsgw_apapi.conf

    [Windows]：
    編輯 C:/INTERSTAGE/F3FMjs2/conf/jsgw_apapi.conf
    加入以下字串

    ApJServMount /servlet ajpv12://localhost:8010/

    [    Solaris]：
    編輯 /opt/FJSVjs2/conf/jsgw_apapi.conf
    加入以下字串

    ApJServMount /servlet ajpv12://localhost:8010/


  2. Setting jsgw.conf (Web Server IIS only)

    [Windows]：
    編輯 C:/INTERSTAGE/F3FMjs2/conf/jsgw.conf
    加入以下字串

    Mount /servlet ID2
    Container ID2 ajpv12://localhost:8010/


  3. 新增 Servlet Container XML。

步驟五(Step 5)：重新啟動 Servlet Service 及 Web Server。

[Windows]：

  1. 停止 Web Server。

    在 NT 服務 視窗中停止 FJapache 服務。

  2. 停止 Servlet Service。

    在 DOS BOX 下切換目錄至 C:\INTERSTAGE\F3FMjs2\bin\
    執行 jssvstop 。

  3. 啟動 Servlet Service。

    在 DOS BOX 下切換目錄至 C:\INTERSTAGE\F3FMjs2\bin\
    執行 jssvstart 。

  4. 啟動 Web Server。

    在 NT 服務 視窗中啟動 FJapache 服務。

[Solaris]：

  1. 重新啟動 Solaris。
  2. 啟動 Servlet Service。

    執行 /opt/FJSVjs2/bin/jssvstart
  3. 啟動 Web Server。

    執行 /opt/FJSVihs/bin/httpd

步驟六(Step 6)：測試 jLIVE™ middle ware。

  1. 開啟 IE ，在 URL 中輸入
    HTTP://localhost
    IE 將會出現 Interstage 範例程式畫面，表示 HTTP Server 服務正常 。
  2. 在 URL 中輸入
    HTTP://localhost/servlet/com.probe.jslive.ProbeServlet/
    IE 將會出現 jLIVE™ middle ware 的管理模組，輸入 ID Password 後，按選Server Configuration 。
  3. 檢視 Authorized ID 及 Authorized concurrent users 是否為所購買的授權數。
  4. 如果出現任何無法啟動或非預期的狀況，請將【Interstage 安裝路徑】【F3FMjs2】【log】目錄下的記錄檔傳至 service@probe.com.tw。

　

　

關於應用程式 deployment 

  1. 由 jLIVE Builder™ 所產生的執行模組包括 *.html 及以下 class 的所有檔案，可安置於 HTTP Server。
    複製至何目錄，須查詢 HTTP SERVER conf 檔。

    [Windows]：
    C:\INTERSTAGE\F3FMihs\conf\httpd.conf。

    [Solaris]：
    /opt/FJSVihs/conf/httpd.conf。

    開啟並尋找 DocumnetRoot 其後所記載的目錄即為 Web Server root directory。

  2. HTTP server 及Interstage 預設是運作於 port 80，所以請檢查應用程式開發時的 middle ware HTTP 是否已改為 80 port。

關於 JDBC driver 

  1. 若只使用於 jLIVE™ 系統，可以直接將 JDBC jar 檔，複製於[Interstage 安裝路徑]\F3FMjs2\webapps\servlet\WEB-INF\lib 中，若是 class 檔則複製至[Interstage 安裝路徑]\F3FMjs2\webapps\servlet\WEB-INF\classes 中。
    例如：
    安置 middle ware 中的 oracle driver ：將 [jslive 授權光碟]\jslive\jre141\oracle_driver 目錄下的所有檔案複製至 【Interstage  安裝路徑】【F3FMjs2】【webapps】【servlet】【WEB-INF】【classes】目錄下。
