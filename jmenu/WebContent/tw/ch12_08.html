LocalHashPut(key,value)

　

　 說明：

Local Hash是jLIVE Builder™ 提供的區域變數，只存在於被定義的原始視窗，其內含值無法被別的視窗參用，離開原始視窗後，該視窗定義的所有Local Hash 都會自動消失。

定義區域變數的方法很簡單，設計師只要在原始視窗任何一個元件、事件、函數中，使用到 LocalHashPut(key,value) 或是 LocalHashGet(key) 函數，假如該索引鍵是第一次被使用，系統會自動造出該區域變數。索引鍵(key)名稱由設計師自行定義，型態為字串，必須用引號括起來，鍵值(value) 則可以是任何形態。

LocalHashPut 函數會將鍵值(value) 存到索引鍵(key) 裡面。
參數： key (索引鍵)、value (鍵值)。
傳回值： value (鍵值)。
範例： LocalHashPut("test" , 111)，將 111 存到索引鍵名稱為 "test"的區域變數中，傳回111。

LocalHashGet(key)

　

　 說明： 獲得區域變數的內含值，作用後該區域變數內含值不會改變。
"key" 為區域變數的索引鍵名稱，若該區域變數不存在，傳回空值。
參數： key (索引鍵)。
傳回值： value (鍵值)。
範例： LocalHashGet("test")，傳回索引鍵名稱為 "test"的區域變數內含值。

說明： 移除區域變數，此函數作用之後，該區域變數消失。
參數： key (索引鍵)。
傳回值： key (索引鍵)。
範例： LocalHashRemove("test")，移除索引鍵名稱為 "test"的區域變數。

StackPush(x)

　

說明：

Stack 是jLIVE Builder™ 提供的全域性的"容器"，可以跨視窗存取資料，是設計師很方便且常用的功能。Stack 的特性是 "先入後出" ，把值壓入 Stack 與取出 Stack 的值需特別留意它的 " 順序性 "。

我們可以把它想像成汽車上常見的一種放硬幣的小盒子， 當我們把硬幣一個一個往裡面塞時，它底下的彈簧會把所有的硬幣緊緊的頂住，使用時就從頂端把硬幣一個一個取出，所以最先塞進去的硬幣會最後才拿出。

"壓入" 時使用 StackPush(x)，"取出"時使用 StackPop()。
參數： x (運算式內含值或常數)。
傳回值： x (運算式內含值或常數)。
範例： StackPush({Customers:CustomerID})，將客戶編號({Customers:CustomerID})壓入Stack。

範例： StackPop()。

HashPut(key,value)

　

　 說明：

Hash是jLIVE Builder™ 提供的全域變數，一經定義，則內含值可在任何視窗參用。

定義 Hash 的方法很簡單，設計師只要在任何視窗任何一個元件、事件、函數中，使用到 HashPut(key,value) 或是 HashGet(key) 函數，假如該索引鍵是第一次被使用，系統會自動造出該全域變數。索引鍵(key)名稱由設計師自行定義，型態為字串，必須用引號括起來，鍵值(value) 則可以是任何形態。

Hash 與 Local Hash 的函數使用方式完全相同，唯一的差異在於前者是全域變數，而後者是區域變數而已。

HashPut 函數會將鍵值(value) 存到索引鍵(key) 裡面。
參數： key (索引鍵)、value (鍵值)。
傳回值： value (鍵值)。
範例： HashPut("userid" , {Employees:EmployeeID})，將 {Employees:EmployeeID}(員工編號) 存到索引鍵名稱為 "userid"的 Hash 全域變數中，傳回{Employees:EmployeeID}。

　

▲Top

　

　

HashGet(key)

　

　 說明： 獲得 Hash 全域變數的內含值，作用後該全域變數內含值不會改變。
"key" 為全域變數的索引鍵名稱，若該全域變數不存在，傳回空值。
參數： key (索引鍵)。
傳回值： value (鍵值)。
範例： HashGet("userid")，傳回索引鍵名稱為 "userid" 的全域變數內含值。

傳回值： key (索引鍵)。
範例： HashRemove("userid")，移除索引鍵名稱為 "userid"的全域變數。

SysHashPut(key,value)

　

　 說明：

SysHash是jLIVE Builder™ 提供的全域變數，一經定義，則內含值可在任何視窗參用。

定義 SysHash 的方法很簡單，設計師只要在任何視窗任何一個元件、事件、函數中，使用到 SysHashPut(key,value) 或是 SysHashGet(key) 函數，假如該索引鍵是第一次被使用，系統會自動造出該全域變數。索引鍵(key)名稱由設計師自行定義，型態為字串，必須用引號括起來，鍵值(value) 則可以是任何形態。

SysHash 與 Hash 函數的特性完全相同，唯一需留意的地方是，jLIVE Builder™ 提供的登入機制與安全控管中，會將使用者的某些資訊儲存在 SysHash 裡面，假如設計師沿用系統的登入機制與安全控管視窗，需避開預設的索引鍵，以免發生不可預期的錯誤。

系統預設的 SysHash 索引鍵詳列如下，建議設計師盡量使用 Hash 全域變數。

"compcode"、"userid"、"username"、"userpass"、"userduty"、"sessionid"

SysHashPut 函數會將鍵值(value) 存到索引鍵(key) 裡面。
參數： key (索引鍵)、value (鍵值)。
傳回值： value (鍵值)。
範例： SysHashPut("userid" , {Employees:EmployeeID})，將 {Employees:EmployeeID}(員工編號) 存到索引鍵名稱為 "userid"的 SysHash 全域變數中，傳回{Employees:EmployeeID}。

SysHashGet(key)

　

　 說明： 獲得 SysHash 全域變數的內含值，作用後該全域變數內含值不會改變。
"key" 為 SysHash 全域變數的索引鍵名稱，若該全域變數不存在，系統會自動造出，並傳回空值。
參數： key (索引鍵)。
傳回值： value (鍵值)。
範例： SysHashGet("userid")，傳回索引鍵名稱為 "userid" 的 SysHash 全域變數內含值。

AssignSysVar(i,x)

　

　 說明： _sysvar  是jLIVE Builder™ 提供的全域變數，編號從1到8，不需由設計師定義，其內含值可在任何視窗直接參用。
AssignSysVar 函數會將鍵值(x) 存到第 i 個全域變數裡面。
參數： i (編號)、x (鍵值)。
傳回值： x (鍵值)。
範例： AssignSysVar( 3  , {Employees:EmployeeID})，將 {Employees:EmployeeID}(員工編號) 存到全域變數 {SysVar_3}。
