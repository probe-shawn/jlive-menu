【Northwind】-- Sample Application

　

Categories

　

視窗名稱 (Form Name)：BASC_CATE
　 　
操作說明 (Specifications)：
　

  • Category資料的維護作業。

  • 【Image Transport】標籤頁示範圖片檔案的管理技巧。

  • 【File Transport】標籤頁示範.HTML檔案的管理技巧。
視窗屬性 (Form Properties)：
　

  • 【事件】【資料來源開啟後】選擇 ini，表示視窗開啟後即執行 ini 事件。
資料來源 (Data Source)：
Categories：

category資料檔
【SQL】：Select * from categories
元件 (Components)：
【tab_cate】

索引標籤元件 (Tab)。
【Picture】

編輯方塊元件 (Edit)。
【資料來源】【資料欄位】連接 Categories / Picture， 記錄圖片檔案名稱。
【HtmlFile】

編輯方塊元件 (Edit)。
【資料來源】【資料欄位】連接 Categories / HtmlFile， 記錄.HTML檔案名稱。
【Image_File】

編輯方塊元件 (Edit)。
初始時空白，將存放新圖片檔案的路徑。
【btn_get_imagepath】

按鈕元件 (button)。
【動作事件】選擇 get_newimage ，按下時將啟動檔案瀏覽訊息方塊(dialog)。

【New_Image】

圖片元件 (image)。
初始時空白，當客戶端(client)輸入新圖片檔案的路徑之後，再以formula將圖片貼上。
【btn_upload_image】

按鈕元件 (button)。
【動作事件】選擇 upload_image ，按下時會將客戶端(Client) 的圖片檔案上傳到主機。

【radio_image】

選擇鈕元件 (Radiogroup)。
【資料來源】

    【選擇鈕】共有Use new file name、Use original file name兩組。以Use new file name為例，

             進入【選擇鈕屬性】【版面】【文字】填"Use new file name"，此為 第

             一個選擇鈕顯示的文字。
    【選取】【條件運算式】填入{radio_image}=1，其意義為當【radio_image】內含

             值為"1"時， 第一個選擇鈕為被選取狀態。 【選取值】【被選時填入內含值】運

             算式填入  1，代表當客戶端(Client)點選第一個選擇鈕時，會將【radio_image】

             內含值更改成  1 。
如果客戶端(Client) 選取"Use new file name"，則新圖片檔案上傳到主機之後， 會將Categories / Picture內含值更新為新圖片的檔案名稱；如果客戶端(Client) 選取"Use original file name"，則新圖片檔案會以Categories / Picture內含值作為檔案名稱，上傳到主機，而Categories / Picture內含值維持不變。
【btn_upload_file】

按鈕元件 (button)。
【動作事件】選擇 upload_file ，按下時會將客戶端(Client) 的.HTML檔案上傳到主機。

【btn_link_oldfile】

按鈕元件 (button)。
【動作事件】選擇 link_oldfile ，按下時會超連結(hyperlink)到Categories / HtmlFile所記錄的.HTML檔案。

【btn_link_newfile】

按鈕元件 (button)。
【動作事件】選擇 link_newfile ，按下時會超連結(hyperlink)到客戶端(Client) 輸入的HTML檔案。

【btn_get_filepath】

按鈕元件 (button)。
【動作事件】選擇 get_newfile ，按下時將啟動檔案瀏覽訊息方塊(dialog)。

【radio_filename】

選擇鈕元件 (Radiogroup)。
【資料來源】

    【選擇鈕】共有Use new file name、Use original file name兩組。

             以Use new file name為例，進入【選擇鈕屬性】【版面】【文字】填"Use new

     file name"，此為 第一個選擇鈕顯示的文字。
    【選取】【條件運算式】填入{radio_filename}=1，其意義為當

           【radio_filename】內含值為"1"時， 第一個選擇鈕為被選取狀態。

    【選取值】【被選時填入內含值】運算式填入  1，代表當客戶端(Client)點選第一個

             選擇鈕時，會將【radio_filename】內含值更改成  1 。
如果客戶端(Client) 選取"Use new file name"，則新HTML檔案上傳到主機之後， 會將Categories / HtmlFile內含值更新為新HTML的檔案名稱；如果客戶端(Client) 選取"Use original file name"，則新圖片檔案會以Categories / HtmlFile內含值作為檔案名稱，上傳到主機，而Categories / HtmlFile內含值維持不變。
【nav_cate】

導覽元件 (Navigator)。
【資料來源】連接 Categories ，反應客戶端(Client) 新增、修改、刪除等等對資料的操作。
【事件】【存檔】選擇 msg_chk_name，表示當客戶端(Client)按存檔按鈕時，不執行內定的存檔動作，而執行msg_chk_name事件。
事件與事件表 (event schema)：
【ini】

執行運算式事件。
【執行】【運算式】有兩組：
1. LocalHashPut("filePath",UncCodePath()$"doc\")，將

    UncCodePath() 接上字串 "doc\" ，再存入區域變數 "filePath"裡。
2. LocalHashPut("imagePath",UncCodePath()$"images\")，將

    UncCodePath() 接上字串 "images\" ，再存入區域變數 "imagesPath"裡。
UncCodePath()是在做檔案管理時，一個非常好用的運算式，Unc全名是Universal Naming Convention，傳回值為 //本機(主機)名稱/應用程式(*.html所在)資料夾分享名稱/class/， 需要注意的是，此運算式run time時會依環境不同而傳回不同的值，詳述如下：


1. 在jLIVE Builder的設計模式下執行：傳回  C:\jLIVE\programs\jNorth\class\
2. 以設計師本機端的html檔執行(ex:c:\jLIVE\programs\jNorth\jNorth.html)：傳回 

    /C:/jLIVE/programs/jNorth/class/
3. 在主機上執行：傳回  //server-2/jNorth/class/     (server-2 為主機名稱)
【msg_chk_name】

出現訊息方塊事件。
【條件】運算式為 LRTrim({CategoryName})=""，意即CategoryName為空值時才會執行此事件。
【否則】選 post_cate ，意即CategoryName不是空值就執行post_cate事件。
【執行】

    【標題】填入Message

    【訊息】運算式填入 "Please input Category Name !!"，執行期此字串會

                     出現在訊息方塊中間

    【選項類別】選確定，執行期將只出現一個確定按鈕。
【後續事件】選 focus_name，意即此事件執行後，接著執行 focus_name  事件。
【focus_name】

元件focus事件。
【執行】【設定focus】勾選CategoryName，會將focus移到CategoryName元件上。
【post_cate】

資料來源之導覽元件事件。
【執行】【資料來源】選Categories，【操作】選存檔，執行Categories的存檔動作。
【get_newimage】

執行運算式事件。
【執行】【運算式】有兩組：
1. SetProp("Image_File","text",

           FileChooser("select file",1," *.gif,*.jpg"))

     FileChooser()執行時會出現檔案選擇視窗，"1"與" *.gif,*.jpg"兩個參數意義是只出現

     " *.gif,*.jpg"檔案，傳回客戶端(client)所選取之檔案路徑名稱。再由SetProp()運算式將

    檔案路徑名稱填入"Image_File"元件。
2. SetProp("New_Image","uncfile",{Image_File})，將元件{Image_File} 的

    內含值，當作圖片元件"New_Image" 的"靜態圖片"之檔案路徑名稱，執行時客戶端

    (client)所選取之圖片，會出現在元件上。
【upload_image】

執行運算式事件。
【條件】運算式為 {radio_Image}=1 and LRTrim({Image_File})<>""，意即Image_File不是空值，而且客戶端(client)選擇 Use new file name 時({radio_Image}=1)，才會執行此事件。
【否則】選 upload_image1 ，意即Image_File是空值 ，或是客戶端(client)選擇 Use original file name 時就執行upload_image1 事件。
【執行】【運算式】有三組：
1. CopyTo({Image_File},
        LocalHashGet("imagePath")$

        {Image_File}[FindLast({Image_File},"\")+1 to

                     Length2({Image_File})])

    CopyTo(source,target)運算式執行時會將檔案(source)拷貝到指定的地方(target)。在此，

    source是客戶端(client)所選取之檔案路徑名稱{Image_File} ，target的路徑存放在區域變

    數 "imagePath" 裡，而{Image_File}[FindLast({Image_File} ,"\" )+1 to Length2({Image_File} )]

    這段運算式目的在於，把{Image_File} 元件最後一個"\"字元的下一位開始(FindLast

    ({Image_File} ,"\" )+1)，到最後一位為止(Length2({Image_File} ))的字串切下來，作為傳

    回值，得到的實際上就是檔案名稱。前後組起來也就成了完整的target的檔案路徑 。
2. SetFieldData("Categories","Picture" ,

               {Image_File}[FindLast({Image_File},"\")+1 to

                            Length2({Image_File})])

    SetFieldData()運算式會把值(檔案名稱)，填入資料欄位("Categories"的"Picture")中。
3. LocalHashPut("postok", DsNavigator("Categories" ,"post" ))，

    DsNavigator("Categories" ,"post" )把運算式2的編輯動作存檔，執行成功傳回1，否則傳回

    0，而區域變數 "postok"則存放其傳回值。
【後續事件】選 upload_success，意即此事件執行後，接著執行 upload_success  事件。
【upload_success】

出現訊息方塊事件。
【條件】運算式為  LocalHashGet("postok")=1 ，意即Categories存檔成功時才會執行此事件。
【執行】

    【標題】填入Message

    【訊息】運算式填入 "Upload file success !"，執行期此字串會出現在訊息

                     方塊中間

    【選項類別】選確定，執行期將只出現一個確定按鈕。
【upload_image1】

執行運算式事件。
【條件】運算式為 LRTrim({Image_File})<>""，意即Image_File不是空值才會執行此事件。
【否則】選 need_file ，意即Image_File是空值時就執行need_file  事件。
【執行】【運算式】有兩組：
1. CopyTo({Image_File},

         LocalHashGet("imagePath")${Categories:Picture})

    把客戶端(client)所選取之檔案路徑名稱{Image_File} ，拷貝到路徑為 "imagePath" 的指定

    資料夾，而且延用舊的檔案名稱({Categories:Picture})。
2. SetProp("image","refresh",1) and

  SetProp("Old_Image","refresh",1)，叫兩個圖片元件"image"與

     "Old_Image"，重新讀入資料(refresh)。
【need_file】

出現訊息方塊事件。
【執行】

    【標題】填入Message

    【訊息】運算式填入 "File name needed !"，執行期此字串會出現在訊息方塊

                     中間

    【選項類別】選確定，執行期將只出現一個確定按鈕。
【get_newfile】

執行運算式事件。
【執行】

    【運算式】SetProp("NEW_FILE","text",

                  FileChooser("select file",1,1))

        FileChooser()執行時會出現檔案選擇視窗，傳回客戶端(client)所選取之檔案路徑名

        稱。再由SetProp()運算式將檔案路徑名稱填入"NEW_FILE" 元件。
【upload_file】

執行運算式事件。
【條件】運算式為 {radio_filename}=1 and LRTrim({NEW_FILE})<>""，意即NEW_FILE不是空值，而且客戶端(client)選擇 Use new file name 時({radio_filename}=1)，才會執行此事件。
【否則】選 upload_file1 ，意即NEW_FILE是空值 ，或是客戶端(client)選擇 Use original file name 時就執行upload_file1 事件。
【執行】【運算式】有三組：
1. CopyTo({NEW_FILE},

         LocalHashGet("filePath")$

        {NEW_FILE}[FindLast({NEW_FILE},"\")+1 to

    source是客戶端(client)所選取之檔案路徑名稱{NEW_FILE} ，target的路徑存放在區域

   變數 "filePath" 裡，而{NEW_FILE}[FindLast({NEW_FILE} ,"\" )+1 to Length2({NEW_FILE}

    )]這段運算式目的在於，把{NEW_FILE}  元件最後一個"\"字元的下一位開始(FindLast

    ({NEW_FILE} ,"\" )+1 )，到最後一位為止(Length2({NEW_FILE} ))的字串切下來，作為

    傳回值，得到的實際上就是檔案名稱。前後組起來也就成了完整的target的檔案路徑 。
2. SetFieldData("Categories","HtmlFile",

              {NEW_FILE}[FindLast({NEW_FILE},"\" )+1 to

                         Length2({NEW_FILE})])

  SetFieldData()運算式會把值(檔案名稱)，填入資料欄位("Categories"的"HtmlFile")中。
3. LocalHashPut("postok", DsNavigator("Categories" ,"post" ))，

    DsNavigator("Categories" ,"post" )把運算式2的編輯動作存檔，執行成功傳回1，否則傳回

    0，而區域變數 "postok"則存放其傳回值。
【後續事件】選 upload_success，意即此事件執行後，接著執行 upload_success  事件。
【upload_file1】

執行運算式事件。
【條件】運算式為  LRTrim({NEW_FILE})<>""，意即NEW_FILE不是空值時才會執行此事件。
【否則】選 need_file ，意即NEW_FILE是空值時就執行need_file 事件。
【執行】

【運算式】CopyTo({NEW_FILE},

              LocalHashGet("filePath")${Categories:HtmlFile})

                      把客戶端(client)所選取之檔案路徑名稱{NEW_FILE}，拷貝到路徑為 "filePath"

                      的指定資料夾，而且延用舊的檔案名稱({Categories:HtmlFile})。
【後續事件】選 upload_success，意即此事件執行後，接著執行 upload_success  事件。
【link_oldfile】

執行超連結事件。
【條件】運算式為  LRTrim({OLD_FILE})<>""，意即OLD_FILE不是空值時才會執行此事件。
【執行】【運算式】HttpCodeBase()$"doc/"${OLD_FILE} ，傳回值為 http://本機(主機)DNS/應用程式(*.html所在)資料夾分享名稱/class/， 需要注意的是，此運算式run time時會依環境不同而傳回不同的值如下：
1. 在jLIVE Builder的設計模式下執行：傳回  C:\jLIVE\programs\jNorth\class\
2. 以設計師本機端的html檔執行(ex:c:\jLIVE\programs\jNorth\jNorth.html)：傳回  

     file:/C:/jLIVE/programs/jNorth/class/
3. 在主機上執行：傳回  http://www.jlive.com.tw/jNorth/class/  
【link_newfile】

執行超連結事件。
【條件】運算式為  LRTrim({NEW_FILE})<>""，意即NEW_FILE不是空值時才會執行此事件。
【執行】【運算式】"file:"${NEW_FILE}。
