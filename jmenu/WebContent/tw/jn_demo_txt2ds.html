Appendix

　

【Northwind】-- Sample Application

　

Insert data from text file into table

　

視窗名稱 (Form Name)：DEMO_TXT2DS
　 　
操作說明 (Specifications)：
　

  • 將文字檔 (txt2ds.txt) 的資料載入Buffered 資料來源 (BUFF_TXT)。

  • 將 Buffered 資料來源 (BUFF_TXT)，依序插入 (insert) 實體表格 (orderchart)。這個流程有多種做法，此視窗示範其中兩種，第一種以資料來源之導覽元件事件，第二種以 NON_SELECT_SQL事件。
資料來源 (Data Source)：
BUFF_TXT：

Buffered 資料來源。
【資料欄位】：Sales、Orderdate、Product、Unitprice、Quantity、Discount，分別對應文字檔 (txt2ds.txt) 的資料欄位。
orderchart：

實體表格的資料來源。
【SQL】：select * From orderchart
元件 (Components)：
【edit_path】

編輯方塊元件 (Edit)。
【資料來源】核取 使用運算式，核取 只在初始化時作用。運算式內容為
  "C:\jLIVE\programs\jNorth\class\doc\txt2ds.txt"， 這是jLIVE Builder 安裝的預設路徑。
【btn_txt2buff】

按鈕元件 (button)。
【動作事件】選擇 txt_2_buff ，按下時將文字檔 (txt2ds.txt) 的資料載入Buffered 資料來源 (BUFF_TXT)。

【tab_txt】

索引標籤元件 (Tab)。
【btn_delall_buff】

按鈕元件 (button)。
【動作事件】選擇 delall_buff ，按下時將 Buffered 資料來源 (BUFF_TXT) 的資料全部刪除。

【btn_dsnav_ins】

按鈕元件 (button)。
【動作事件】選擇 buff_top1 ，按下時以資料來源之導覽元件事件，將 Buffered 資料來源 (BUFF_TXT)，依序插入 (insert) 實體表格 (orderchart)。

【btn_sql_ins】

按鈕元件 (button)。
【動作事件】選擇 buff_top2 ，按下時以 NON_SELECT_SQL 事件，將 Buffered 資料來源 (BUFF_TXT)，依序插入 (insert) 實體表格 (orderchart)。

【grid_buff】

資料表元件 (Grid)。
【資料來源】連接 BUFF_TXT。
【btn_delall_ds】

按鈕元件 (button)。
【動作事件】選擇 delall_ds ，按下時將實體表格 (orderchart) 的資料全部刪除。

【tot_rec】

編輯方塊元件 (Edit)。
 【資料來源】核取 使用運算式。運算式內容為 {orderchart:TotalRecord()}，顯示資料來源(orderchart) 的資料筆數。
【grid_table】

資料表元件 (Grid)。
【資料來源】連接 orderchart。
事件與事件表 (event schema)：
【txt_2_buff】

載入文字檔至 Buffer 資料來源事件。
【條件】運算式為 1 or SetProp("tab_txt","selected",1)。目的在於切換標籤頁到第一頁。
【文字】
核取 Tab Delimited ，表示文字檔 (txt2ds.txt) 以 "tab" 為分隔符號。
運算式為 {edit_path}，這是文字檔的路徑 (path)。
定義文字記錄 建立六個欄位，這是文字檔的欄位格式。
【Buffer 資料來源】選 BUFF_TXT。
資料欄位轉換對應表 定義文字檔與 Buffer 資料來源再資料轉換時，所對應的欄位關係，例如第一個文字欄位 "fld1" 內含值，將填入 BUFF_TXT 的 Sales 欄位。
這個事件是一個"迴圈事件"，執行時會依序把所有的文字檔資料，全部填入 Buffer 資料來源事件。
【delall_buff】 資料來源之導覽元件事件。
【執行】【資料來源】選 BUFF_TXT，【操作】選全刪 (delete all)，刪除Buffer 資料來源 BUFF_TXT 的全部記錄。
【delall_ds】 執行 NON_SELECT_SQL 命令事件。
【執行】【資料庫】選 Northwind
【Non-Select SQL 命令】為 delete from orderchart
刪除資料來源 orderchart 的全部記錄。
【後續事件】選 reopen_ds。
【reopen_ds】 資料來源之導覽元件事件。
【執行】【資料來源】選 orderchart，【操作】選重開 (reopen)，因為前一個事件雖然已將 orderchart 的資料全部刪除，但是執行Non-Select SQL 命令之後，視窗元件 (grid_table) 並不會即時反應資料表格的異動，因此重開 (reopen) 資料來源，以將 grid 上的資料將全部清空。
【buff_top1】 資料來源之導覽元件事件。
【條件】運算式為 not {BUFF_TXT:IsEmpty()}，意即 BUFF_TXT 有記錄時才會執行此事件。
【執行】【資料來源】選 BUFF_TXT，【操作】選首筆 (First)，將記錄指標調到 BUFF_TXT 的第一筆。
【後續事件】選 ins_ds。
【ins_ds】 資料來源之導覽元件事件。
【執行】【資料來源】選 orderchart，【操作】選新增 (insert)，執行後將新增一筆記錄。
【後續事件】選 set_fld。
【set_fld】 設定資料來源記錄之欄位內含值事件。
【執行】【資料來源】選 orderchart，【資料欄位】依序將 BUFF_TXT 的欄位內含值填入 orderchart。
【後續事件】選 post_ds。
【post_ds】 資料來源之導覽元件事件。
【執行】【資料來源】選 orderchart，【操作】選存檔 (post)。
【後續事件】選 next_buff1。
【next_buff1】 資料來源之導覽元件事件。
【條件】運算式為 not {BUFF_TXT:Eof()}，意即 BUFF_TXT 的記錄指標尚未到達最後一筆時才會執行此事件。
【否則】選 msg_done。
【執行】【資料來源】選 BUFF_TXT，【操作】選下筆 (next)，將 BUFF_TXT 的記錄指標下移一筆。
【後續事件】選 ins_ds。上述4個事件構成一個迴圈，將 BUFF_TXT 的記錄全部轉入資料來源 orderchart。
【msg_done】

出現訊息方塊事件。
【條件】運算式為 1 or SetProp("tab_txt","selected",2)，目的在於自動切換到標籤元件的第二頁，與事件執行的條件無關。
【執行】

    【標題】填入Message

    【訊息】運算式填入 "The operaton of insertion completed !\n"$"There were "${BUFF_TXT:TotalRecord()}$" records added to table."，"\n" 為換行符號。

   【選項類別】選確定，執行期將只出現一個確定按鈕。
【buff_top2】 資料來源之導覽元件事件。
【條件】運算式為 not {BUFF_TXT:IsEmpty()}，意即 BUFF_TXT 有記錄時才會執行此事件。
【執行】【資料來源】選 BUFF_TXT，【操作】選首筆 (First)，將記錄指標調到 BUFF_TXT 的第一筆。
【後續事件】選 ins_sql。
【ins_sql】 執行 NON_SELECT SQL 命令事件。
【執行】【資料庫】選 Northwind
【Non-Select SQL 命令】為
   Insert into orderchart (Sales,Orderdate,Product,
               UnitPrice,Quantity,Discount)
   Values (?,?,?,?,?,?)
         這是標準 SQL-92 的Insert 命命，不同的是在 Values 後面以 6 個參數來將內
　　含值帶入。
　　參數 1類別：VARCHAR，運算式：{BUFF_TXT:Sales}。
　　參數 2類別：TIMESTAMP，運算式：{BUFF_TXT:Orderdate}。
　　參數 3類別：VARCHAR，運算式：{BUFF_TXT:Product}。
　　參數 4類別：INTEEGER，運算式：{BUFF_TXT:Unitprice}。
　　參數 5類別：INTEGER，運算式：{BUFF_TXT:Quantity}。
　　參數 6類別：DECIMAL，運算式：{BUFF_TXT:Discount}。
【後續事件】選 reopen_ds。
【next_buff2】 資料來源之導覽元件事件。
【條件】運算式為 not {BUFF_TXT:Eof()}，意即 BUFF_TXT 的記錄指標尚未到達最後一筆時才會執行此事件。
【否則】選 reopen_ds1。
【執行】【資料來源】選 BUFF_TXT，【操作】選下筆 (next)，將 BUFF_TXT 的記錄指標下移一筆。
【後續事件】選 ins_sql。上述 2個事件構成一個迴圈，將 BUFF_TXT 的記錄全部轉入資料來源 orderchart。
【reopen_ds1】 資料來源之導覽元件事件。
【執行】【資料來源】選 orderchart，【操作】選重開 (reopen)，執行Non-Select SQL 命令之後，視窗元件 (grid_table) 並不會即時反應資料表格的異動，因此重開 (reopen) 資料來源。
【後續事件】選 msg_done。
