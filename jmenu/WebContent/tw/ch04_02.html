4.3 工具 (tool)

應用程式輸出 (application export)

應用程式輸入 (application import)

應用程式線上更新 (application online update)

　

jLIVE Builder™ 提供應用程式模組輸出 (export)、輸入 (import) 及線上更新 (online update) 等應用程式的保存、載入及維護方式。應用程式輸出 (export) 是將該專案應用程式的所有資源 (resource)，包括所有設定、 XML、圖檔、原始程式碼等等，壓縮 (jar) 成保存檔 (*.jlv)。利用應用程式輸入(import) ，可以將整個專案應用程式的所有資源 (resources)，載入到任何其他的 jLIVE Builder™ 作業平台。
　

應用程式線上更新 (application online update) 是 jLIVE Builder™ 特有的功能，也是最方便、最有效率的程式維護、更新機制。jLIVE Builder™ 所開發的應用程式是分散性的程式模組 (distributed runtime module)，集中於 Web 主機，當應用程式有任何的修改，甚至只是簡單的畫面元件位移，過去的主從架構 (client/server) 或非分散性程式模組，需要重新編譯 (compile) 整個應用程式成 *.exe 檔，再將 *.exe 檔下載至每一個客戶端 (client)，jLIVE Builder™ 由於是分散性的程式模組 (distributed runtime module)，只要編譯 (compile) 更新或新增的視窗即可，再透過應用程式線上更新 (application online update) 將該視窗上載 (upload) 至 Web 中即可，每個客戶端 (client) 在參照到該視窗模組時即會自行下載更新。

　

　

輸出 (export)

　

選擇輸出的模組名稱、路徑及輸出項目。

  • 存成模組名稱：輸出模組名。系統會自動加上 .jlv 的附屬名稱。

  • 輸出至路徑：輸出的路徑。

  • 選項：

    • 輸出 XML：輸出 XML 檔案。檔案位置在應用程式主目錄下的子目錄 "xml"。　

    • 輸出 Java 程式碼：檔案位置在應用程式主目錄下的子目錄 "source"。　

    • 輸出編譯 class：檔案位置在應用程式主目錄下的子目錄 "class"。　

    • 輸出技術文件：檔案位置在應用程式主目錄下的子目錄 "doc"。　

    • 輸出圖片：檔案位置在應用程式主目錄下的子目錄 "\class\images"。　

    • 輸出 預載圖片：檔案位置在應用程式主目錄下的子目錄 "\class\preloadimage"。　

    • 輸出 Doc：檔案位置在應用程式主目錄下的子目錄 "\class\doc"。　

    • 輸出 Audio：檔案位置在應用程式主目錄下的子目錄 "\class\audio"。　

    • 輸出 Video：檔案位置在應用程式主目錄下的子目錄 "\class\video"。　

    • 輸出執行模組：檔案位置在應用程式主目錄的 *.html 和 *.jar 檔案。"*" 代表應用程式名。

線上更新 (online update)

　

當應用程式已經佈建 (deploy) 完成之後，只要有網際網路連線就可以利用此功能更新程式。 使用線上更新功能必須先將程式所在的資料夾分享權限給 jLIVE™ middle ware 所在的主機，如下圖的分享路徑為 Root。 按選【線上更新】後，系統會詢問目標 jLIVE™ middle ware  所在的 URL，使用者可以選取已存在的 URL 或自行輸入。每次線上更新成功傳輸後，系統會自動記錄該次所使用的 URL 及 分享路徑，再次使用線上更新時，即可自選單中直接選取。

　

　

  • 分享路徑 (UNC) : 應用程式所在的目錄，必須分享可讀寫給應用程式伺服器 (jLIVE™ middle ware 所在的主機)。
    　

  • Html & Applet :  

    • Html : 列示應用程式的 html 檔案，勾選檔案即會執行更新。如果是第一次部署應用程式的話，就一定要選擇此檔案。
      　

    • jlive.jar : jlive.jar 是 jLIVE Builder™ 執行期模組 (runtime class)，當使用者 (client) 第一次執行系統時，會先從應用伺服器 (application server) 下載，一旦下載過，以後執行就不需要再下載。除非 jLIVE Builder™ 有更新版本時才需要將新版 jlive.jar 上傳至應用伺服器 (application server)。
      　

    • jimage.jar : jimage.jar 是 jLIVE Builder™ 先將應用程式中常用的圖形壓縮成 jar 檔，在使用者第一次執行程式時，先行下載到使用者端 (client)，除非 jimage.jar 有更新，否則只下載一次，此一機制能加快應用程式圖檔下載速度。jimage.jar 是將 "\{application location}\class\preloadimage" 目錄中的圖形檔壓縮起來，所以當目錄堛犒洇峖釦騝s時，必需一併上傳至伺服器 (web server)。
      　

    • classes : 在這個區域裡的 class 是指非視窗 (form) 和 報表 (report) 的 class。通常這些 class 是屬於 applet 的 class ，或者是曾經使用過但是已經更名的 class 。

　

  • 視窗&報表 : 列示所有的視窗 (form) 和報表 (report) 的 class 。可以選擇 全選 或是 全不選，也可以逐一選擇。

　

  • 提交 : 線上更新功能必須要有應用程式管理者的帳號和密碼，所以當按下提交鈕時，會出現下圖的對話框，只有輸入正確的帳號密碼，才能上傳更新模組。
