BEA WebLogic™ 安裝 jLIVE™ middle ware (setup jLIVE™ middle ware)  

　

步驟一 (Step 1)：安裝 BEA WebLogic platform。

  1. 測試版的網址：http://commerce.bea.com/downloads/products.jsp。

　

步驟二 (Step 2)：建立一個新的 WebLogic Domain。

  1. 選擇執行選單【開始】【程式集】【BEA WebLogic Platform7.0】【Domain Configuration Wizard】進入精靈視窗。

  2. Choose Domain Type And Name：選擇建立 WLS Domain 的 template，並命名為  jlive_domain。

  3. Choose Sever Type：選擇 Single Server (Standalone Server)

  4. Choose Domain Location：以預設路徑即可。

  5. Configure Standalone/Administrator Server：
    Server Name：jlive_server
    Server Listen Address：
    Server Listen Port：80 (或 8080)
    Server  SSL Listen Port：8081 

  6. Create Administrative User：
    User Name：probe_admin
    Password：probe_admin
    Verify Password：probe_admin

  7. Install Server as Windows Service：No

  8. Create Start Menu Entry For Server：Yes

  9. Configiguration Summary：按 Create 鍵，建立 Domain。

步驟三 (Step 3)：修改 web.xml。

  1. 建立 Domain 後，在 bea\user_projects\ 目錄下會新增一 jlive_domain 目錄。
  2. 以 Notepad 開啟 bea\user_projects\jlive_domain\applications\DefaultWebApp\WEB-INF\web.xml。
  3. 將 web.xml 修改為以下：
    <?xml version="1.0" ?>
    <!DOCTYPE web-app PUBLIC "-//Sun Microsystems, Inc.//DTD Web Application 2.3//EN" "http://java.sun.com/dtd/web-app_2_3.dtd">

    <web-app>
        <servlet>
            <servlet-name>
                ProbeServlet
            </servlet-name>
            <servlet-class>
                com.probe.jslive.ProbeServlet
            </servlet-class>
        </servlet>
        <servlet-mapping>
            <servlet-name>
                ProbeServlet
            </servlet-name>
            <url-pattern>
                /servlet/*
            </url-pattern>
        </servlet-mapping>
        <servlet-mapping>
            <servlet-name>
                ProbeServlet
            </servlet-name>
            <url-pattern>
                /servlet/com.probe.jslive.ProbeServlet/*
            </url-pattern>
        </servlet-mapping>
    </web-app>

步驟四(Step 4)：建立 jLIVE Builder middle ware。

  1. 在上述 WEB-INF 目錄下新增兩個目錄 \lib\ 及\classes\。
  2. 將 下列 jar file 複製到 \lib\ 目錄中
    • jslive.jar (jLIVE middle ware)
    • mailapi.jar (SMTP mail api)
    • mysql-connector-java-2.0.14-bin.jar(MySQl driver) 
    • Sprinta2000.jar (MSSQL Driver)
  3. 將以 class 存在的 library 檔複製到 \classes\ 目錄中，例如 oracle driver。

步驟五(Step 5)：建立 httpd 的應用程式模組。

  1. 將由 jLIVE Builder™ 所產生的應用程式的目錄下，啟動程式的 *.html 及class 目錄下所有檔案，複製到 bea\user_projects\jlive_domain\applications\DefaultWebApp\下 。
  2. 以 jLIVE Builder Sample 為例：將 jLIVE\programs\jNorth\jNorth.html 及jLIVE\prograqms\jNorth\class\ 目錄及以下所有檔案複製到 bea\user_projects\jlive_domain\applications\DefaultWebApp\下 。

步驟六(Step 6)： Java™ 版本 ，jLIVE Builder™ middle ware 需要 j2sdk 1.4.0 以上版本。

  1. 修改 bea\weblogic700\server\bin\startWLS.cmd，將 set JAVA_HOME=c:\bea\jdk131_03 改為 set JAVA_HOME=c:\j2sdk1.4.1_01(安裝的 1.4.0 版本以上的 Java 目錄) 。
  2. 將修改完成的檔案另存成 bea\weblogic700\server\bin\startWLS_jlive.cmd。
  3. 修改 bea\user_projects\jlive_domain\startWebLogic.cmd，將檔案最後段的 call "c:\bea\weblogic700\server\bin\startWLS.cmd" 改為 "c:\bea\weblogic700\server\bin\startWLS_jlive.cmd"。

步驟七(Step 7)： Security Policy 。

  1. 修改 Security Policy file 

　

步驟八(Step 8)： 啟動 WebLogic Server 。

  1. 選擇執行選單【開始】【程式集】【BEA WebLogic Platform7.0】【User Projects】【jlive_domain】【Start Server】
  2. 在啟動的 DOS BOX 中輸入 username ：probe_admin 及 password：probe_admin。
  3. 在 DOS BOX 中，WebLogic 會出現 <Server started in RUNNING mode> 表示 WebLogic 以啟動。

步驟九(Step 9)： 測試 jLIVE Builder middle 及 應用程式 。

  1. 開啟 Browser， 輸入URL ：http//localhost (或 :8080)/servlet/com.probe.jslive.ProbeServlet/，Browser 將顯示 Probe Application Server Administrator 的登入畫面，表示 jLIVE middle ware 以正常啟動。
  2. 開啟 Browser， 輸入URL ：http//localhost (或 :8080)/jNorth.html ，Browser 將啟動 jNorth 應用程式。
