【Northwind】-- Sample Application

　

 Sales Information by Month and Customers

　

視窗名稱 (Form Name)：MONT_CUST
　 　
操作說明 (Specifications)：
　

  • 客戶當年度每個月份的訂單數量、採購金額、單張訂單平均採購額資料。
資料來源 (Data Source)：
customers：

客戶資料，作為 combobox 關連的資料來源。
【SQL】：Select CustomerID,CompanyName  From customers
Monthly_Sales：

SQL資料來源，客戶採購資料。
【SQL】：
   select sum(A.UnitPrice*A.Quantity*(1-A.Discount))

          as subtot,
          Month(B.orderdate) as orderdate,

          sum(1) as orders
   From   orders B left outer join orderdetails A
          on A.orderid=B.orderid 
          left outer join customers C
          on B.customerid=C.customerid 
   where  B.customerid=?
   group  by orderdate 
   order  by orderdate
        參數類別 "VARCHAR"，運算式為  {combo_Name}，查詢客戶採購資料。
元件 (Components)：
【combo_Name】

下拉式清單方塊元件 (ComboBox)。
【關連資料來源】【資料欄位】連接 customers/customerID (客戶編號)。
【被選時移至關連資料來源記錄】核取此項。
【顯示】運算式為 {customers:CompanyName} (客戶名稱)。
【關連資料來源初始化時清除選擇】核取此項，則視窗開啟時，此元件為空值。
【btn_trig】

按鈕元件 (button)。
【動作事件】選擇 trig_month ，按下時觸發 "Monthly_Sales" 資料來源 。

【grid_sales】

資料表元件 (Grid)。
【資料來源】連接 Monthly_Sales。
【欄位】【資料表元件欄位】 的最後一個欄位 "Avg. per order" 為計算欄位，運算式為
   if   {Monthly_Sales:orders} <> 0

   then {Monthly_Sales:subtot}/{Monthly_Sales:orders}

   else 0

        如果該月訂單數量 > 0，則計算單張訂單平均採購額，否則填 0。

事件與事件表 (event schema)：
【trig_month】

觸發資料來源事件。
【執行】【資料來源】選 Monthly_Sales。
【SQL】為：
   select sum(A.UnitPrice*A.Quantity*(1-A.Discount))

          as subtot,
          Month(B.orderdate) as orderdate,

          sum(1) as orders
   From   orders B left outer join orderdetails A
          on A.orderid=B.orderid 
          left outer join customers C
          on B.customerid=C.customerid 
   where  B.customerid=?
   group  by orderdate 
   order  by orderdate
       參數類別 "VARCHAR"，運算式為  {combo_Name}，依所選擇的客戶，

       重新查詢其採購資料。
