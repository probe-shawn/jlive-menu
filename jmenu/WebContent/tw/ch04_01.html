4.2 共同開發群組 (work group)

重新掃描群組 (rescan workgroup)

整批複製到本機 (batch remote to local)

　

jLIVE Builder™ 採取對等式的工作群組 (workgroup) 專案協同開發模式。以相同的專案名稱即時共享視窗模組的方式，在程式規格擬定(specification)、設計(design)、除錯(debug)、測試(testing) 等各階段，皆能統合各相關協同開發的工作群組的最新程式模組，即時佈建執行(deploy and execution)。專案的管理人可依系統需求，細分各工作群組之系統範圍，授予相關參照電腦之存取鍵值 (access key)；各工作群組電腦 (workgroup) 隨時掃描相關群組之最新參照模組 (reference)；專案管理人可以隨時監視 (monitoring) 開發進度及測試 (testing) 整體應用程式執行的協同作業狀況，在程式開發階段即可修正實際執行的缺失。

　

使用工作群組 (workgroup) 需要分享jLIVE Builder™ 安裝目錄及應用程式所在的目錄，例如：當應用程式位於 " C:\jLIVE\programs\jNorth\ " 則必須分享 jNorth 目錄和 jLIVE 安裝目錄給工作群組成員。

　

　

重新掃描群組 (rescan workgroup)

　

請先標示該應用程式節點，然後按滑鼠右鍵。這時會出現三個選項，請點選【重新掃描群組】，在此之前必須已經輸入群組成員的 IP/HostName 和存取鍵 (access key)，否則無法掃描。 開始掃描群組時，系統會詢問是否詳列掃瞄結果，如果系統龎大且非第一次掃瞄，建議不要核選詳列掃瞄結果，否則結果報告會很龎大。由於視窗節點是樹狀結構，系統會先行比對結構，其他所有工作組員的視窗節點樹狀結構，會以聯集 (union) 的型式建立在本機視窗節點。同樣的視窗名稱，位於不同節點目錄，系統會視為不合法，並記錄於結果報告中。

  • 掃描結束後，如果有任何異常狀況，將會出現結果報告(scan workgroup report) ，例如：已存在本機或重覆出現在其他工作群組的視窗模組，以及無法連結的工作群組。

  • 掃描結束後，在【視窗】節點內，會出現粉紅色的視窗名稱節點，表示這些視窗是參照其他的群組，而本機並沒有相同的視窗。屬於粉紅色的視窗節點，設計師可以在視窗設計時，參照引用這些視窗，就像這些視窗是本機上所開發的一樣；也可以開啟這些視窗，進入設計模式，檢視所有的設計細節；亦可以改變設計設定，做測試執行，只是在無法儲存所做的改變。如果想將這些視窗，納入本機應用程式之內或做永久的修改，必須先將它複製搬移至本機 (move to local)。

　

  • 如果要將遠端的模組複製到本機，須先標示此視窗節點，然後按下滑鼠右鍵，選擇【複製到本機】就可以將它複製到本機的應用系統中。 如果視窗模組的 XML 檔，已自行複製至本機或希望引用已存在本機的 XML ，使用者按選【參照本機 XML】後，系統將會於預設 XML 目錄搜尋，如果存在則引用本機的 XML 且將節點文字顏色改為黑色，標示為本機節點。

整批複製 (batch remote to local)

  • 整批複製精靈，可以選擇所要複製的模組，或是全選所有的模組，複製到本機。
