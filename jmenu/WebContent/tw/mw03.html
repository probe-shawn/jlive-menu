Interstage™ 安裝 jLIVE™ middle ware (setup jLIVE™ middle ware)  

　

安裝前 (Before setup Interstage)：請先按 [jslive 授權光碟]\jslive\ Middleware 快速安裝.doc ，安裝 middle ware keypro 所需程式模組，否則 jLIVE middle ware 將會自行調整為 light version for development only，屆時只能服務一個線上使用者。

　

　

　

　

步驟一 (Step 1)：安裝 Fujitsu Interstage ™。  

  1. 以 Easy installation 或 Custom installation 安裝後所預設的 Web server 並不相同。

  2. 以下步驟係以 Easy installation 安裝為示範，Custom installation 若使用不同的 Web server 請查閱 Interstage 文件，以相對應所應設定值輸入。

　

步驟二 (Step 2)： 設定 Web Servlet Gateway。

  1. Setting httpd.conf

    1. [Windows] ：
      使用 Interstage HTTP Server
      編輯 [c:\INTERSTAGE]\F3FMihs\conf\httpd.conf
      在 httpd.conf 最後加入以下兩行字串

      LoadModule jsvlt_module C:/INTERSTAGE/F3FMjs2/gateway/jsgw_apapi_is.dll
      include C:/INTERSTAGE/F3FMjs2/conf/jsgw_apapi.conf

      [      Solaris] ：
      使用 Interstage HTTP Server
      編輯 /opt/FJSVihs/conf/httpd.conf
      在 httpd.conf 最後加入以下兩行字串

      LoadModule jsvlt_module /opt/FJSVjs2/gateway/jsgw_apapi.so
      include /opt/FJSVjs2/conf/jsgw_apapi.conf 


    2. 使用 InfoProvider Pro
      參閱 j2ee.pdf -- Chapter 9 (9-6)。

    3. 使用 Internet Information Server
      參閱 j2ee.pdf -- Chapter 9 (9-8)。

    4. 使用 iPlanet Web Server
      參閱 j2ee.pdf -- Chapter 9 (9-8)。

  2. 如果 Servlet Container 與 Web Server 不在同一主機，其設定請參閱 j2ee.pdf -- Chapter 9 (9-10)。

　

步驟三 (Step 3)：設定 Servlet Gateway Definition。

  1. Setting jsgw_apapi.conf

    [Windows]：
    編輯 C:/INTERSTAGE/F3FMjs2/conf/jsgw_apapi.conf
    加入以下字串

    ApJServMount /servlet ajpv12://localhost:8010/

    [    Solaris]：
    編輯 /opt/FJSVjs2/conf/jsgw_apapi.conf
    加入以下字串

    ApJServMount /servlet ajpv12://localhost:8010/


  2. Setting jsgw.conf (Web Server IIS only)

    [Windows]：
    編輯 C:/INTERSTAGE/F3FMjs2/conf/jsgw.conf
    加入以下字串

    Mount /servlet ID2
    Container ID2 ajpv12://localhost:8010/


  3. 新增 Servlet Container XML。

    [Windows]：
    複製 C:/INTERSTAGE/F3FMjs2/conf/sample_jscontainer.xml 並命名為 probe.xml。
    編輯 probe.xml 並更改紅色標示值

    <Server>
    <Logger name="std_log" path="log\sample-std.log" backup="true" timestamp="true" />
    <Logger name="tc_log" path="log\sample-container.log" backup="true" timestamp="true" />
    <Logger name="servlet_log" path="log\sample-servlet.log" backup="true" timestamp="true" />
    <ContextManager workDir="work\servlet" clientSession="permanent" version="5.0" >
    <!-- Don't change the following descriptions. -->
    <ContextInterceptor className="com.fujitsu.interstage.jservlet.tomcat.context.DefaultCMSetter" />
    <ContextInterceptor className="com.fujitsu.interstage.jservlet.tomcat.context.WorkDirInterceptor" />
    <ContextInterceptor className="com.fujitsu.interstage.jservlet.tomcat.context.WebXmlReader" />
    <ContextInterceptor className="com.fujitsu.interstage.jservlet.tomcat.context.LoadOnStartupInterceptor" />
    <RequestInterceptor className="com.fujitsu.interstage.jservlet.tomcat.request.SimpleMapper" />
    <RequestInterceptor className="com.fujitsu.interstage.jservlet.tomcat.request.SessionInterceptor" />
    <RequestInterceptor className="com.fujitsu.interstage.jservlet.tomcat.request.SecurityCheck" />
    <RequestInterceptor className="com.fujitsu.interstage.jservlet.tomcat.request.FixHeaders" />
    <!-- Don't change the above-mentioned descriptions. -->
    <!-- Don't change the following descriptions. -->
    <Connector className="com.fujitsu.interstage.jservlet.tomcat.service.PoolTcpConnector">
    <Parameter name="handler" value="com.fujitsu.interstage.jservlet.tomcat.service.connector.Ajp12ConnectionHandler" />
    <!-- Don't change the above-mentioned descriptions. -->
    <Parameter name="ipaddress" value="localhost"/>
    <Parameter name="port" value="8010"/>
    </Connector>
    <!-- Don't change the following descriptions. -->
    <Context path="" docBase="C:\INTERSTAGE\F3FMjs2\webapps_sys\ROOT" />
    <!-- Don't change the above-mentioned descriptions. -->
    <TagPooling updateInterval="1800" idle="3600" />
    <!-- Add the definition of Web Application under this. -->
    <Context path="/servlet" docBase="C:\INTERSTAGE\F3FMjs2\webapps\servlet" dirList="false" tagPooling="true" />
    </ContextManager>

    </Server>


    [Solaris]：
    複製 /opt/FJSVjs2/conf/sample_jscontainer.xml 並命名為 probe.xml。

  4. Setting jswatch.conf -- JServlet environment definition file

    [Windows]：
    編輯 C:/INTERSTAGE\F3FMjs2/conf/jswatch.conf
    加入以下設定 (可以複製已存在的設定並更改標示藍色值)
    servlet.ipaddress=localhost
    servlet.port=8010
    servlet.log=true
    servlet.containerconf=C:\INTERSTAGE\F3FMjs2\conf\probe.xml
    servlet.bin=C:\j2sdk1.4.1_01\bin\java.exe
    servlet.env=CLASSPATH=C:\j2sdk1.4.1_01\lib\tools.jar
    servlet.env=CLASSPATH=C:\INTERSTAGE\F3FMjs2\classes\jsboot.jar
    servlet.env=CLASSPATH=C:\INTERSTAGE\F3FMjs2\classes\servlet.jar
    servlet.env=CLASSPATH=.\
    servlet.env=CLASSPATH=C:\INTERSTAGE\F3FMjs2\webapps\servlet\WEB-INF\classes
    servlet.env=CLASSPATH=C:\INTERSTAGE\ODWIN\etc\class\ODjava4.jar
    servlet.env=CLASSPATH=C:\INTERSTAGE\ejb\lib\fjcontainer32.jar
    servlet.env=CLASSPATH=C:\INTERSTAGE\j2ee\lib\isj2ee.jar
    servlet.env=PATH=C:\INTERSTAGE\j2ee\bin
    servlet.env=SystemRoot=C:\WINNT\
    servlet.bin.parameters=-Xrs -Dcom.fujitsu.interstage.jservlet.j2ee=true
    -Dorg.omg.CORBA.ORBClass=com.fujitsu.ObjectDirector.CORBA.ORB
    -Dorg.omg.CORBA.ORBSingletonClass=com.fujitsu.ObjectDirector.CORBA.SingletonORB
    -Djavax.rmi.CORBA.StubClass=com.fujitsu.ObjectDirector.rmi.CORBA.StubDelegateImpl
    -Djavax.rmi.CORBA.UtilClass=com.fujitsu.ObjectDirector.rmi.CORBA.UtilDelegateImpl
    -Djavax.rmi.CORBA.PortableRemoteObjectClass=com.fujitsu.ObjectDirector.rmi.CORBA.PortableRemoteObjectDelegateImpl


    [Solaris]：
    編輯 /opt/FJSVjs2/conf/jswatch.conf
    加入以下設定 (可以複製已存在的設定並更改標示藍色值)
    servlet.ipaddress=localhost
    servlet.port=8010
    servlet.log=true
    servlet.containerconf=/opt/FJSVjs2/conf/probe.xml
    servlet.bin=/usr/j2sdk1.4.1_02/bin/java
    servlet.bin.parameters=-Dcom.fujitsu.interstage.jservlet.j2ee=true
    servlet.env=CLASSPATH=/usr/j2sdk1.4.1_02/lib/tools.jar
    servlet.env=CLASSPATH=/opt/FJSVjs2/classes/jsboot.jar
    servlet.env=CLASSPATH=/opt/FJSVjs2/classes/servlet.jar
    servlet.env=CLASSPATH=/opt/FJSVjs2


　

步驟四(Step 4)：安裝 jLIVE middle ware service。jLIVE middle ware 是 Sun Java 1.41 版本。

  1. 安裝 Java 1.41_x 版本。
  2. 加入 WEB APPLICATION。

    [Windows]：
    在 C:\INTERSTAGE\F3FMjs2\webapps\ 目錄下新增以下目錄

    [Solaris]：
    在 opt/FJSVjs2/webapps/ 目錄下新增以下目錄

    servlet
         └ WEB-INF
                 ├classes
                 └lib

  3. 加入 web.xml。

    [Windows]：
    在C:\INTERSTAGE\F3FMjs2\webapps\servlet\WEB-INF\ 下，以文字檔輸入以下內容，並命名為 web.xml

    [Solaris]：
    在 opt/FJSVjs2/webapps/servlet/WEB-INF/ 下，以文字檔輸入以下內容，並命名為 web.xml
    <?xml version="1.0" encoding="UTF-8"?>
    <!DOCTYPE web-app PUBLIC '-//Sun Microsystems, Inc.//DTD Web Application 2.2//EN' 'http:
    /java.sun.com/j2ee/dtds/web-app_2_2.dtd'>
    <web-app>
    <display-name>ProbeServlet</display-name>
    <distributable />
    <servlet>
    <servlet-name>ProbeServlet</servlet-name>
    <display-name>ProbeServlet</display-name>
    <servlet-class>com.probe.jslive.ProbeServlet</servlet-class>
    </servlet>
    <servlet-mapping>
    <servlet-name>ProbeServlet</servlet-name>
    <url-pattern>/*</url-pattern>
    </servlet-mapping>
    <servlet-mapping>
    <servlet-name>ProbeServlet</servlet-name>
    <url-pattern>/com.probe.jslive.ProbeServlet/*</url-pattern>
    </servlet-mapping>
    <session-config>
    <session-timeout>30</session-timeout>
    </session-config>
    </web-app>

    亦可將【jslive 授權光碟】\jslive\patch\interstage\web.xml 直接置入。

    　

  4. 加入 lib
    將【jslive 授權光碟】\jslive\jre141\lib\ 目錄下所有檔案複製至 

    [Windows]：
    C:\INTERSTAGE\F3FMjs2\webapps\servlet\ WEB-INF\lib\ 

    [Solaris]：
    /opt/FJSVjs2/webapps/servlet/WEB-INF/lib/


    加入檔案如下
    • jslive.jar
    • mail.jar
    • mailapi.jar
    • imap.jar
    • pop3.jar
    • smtp.jar
    • mysql-connector-java-2.0.14-bin.jar
    • Sprinta2000.jar

    　

  5. 加入 classes
    將【jslive 授權光碟】\jslive\jre141\classes\ 目錄下所有檔案複製至 

    [Windows]：
    C:\INTERSTAGE\F3FMjs2\webapps\servlet\ WEB-INF\classes\ 

    [Solaris]：
    opt/FJSVjs2/webapps/servlet/WEB-INF/classes/


  6. 加入 SymfoWARE JDBC driver
    將 解壓後的 symfoWARE driver 複製至 

    [Windows]：
    C:\INTERSTAGE\F3FMjs2\webapps\servlet\ WEB-INF\classes\ 

    [Solaris]：
    /opt/FJSVjs2/webapps/servlet/WEB-INF/classes/

  7. 加入 Oracle JDBC driver
    將 解壓後 Oracle driver 複製至 

    [Windows]：
    C:\INTERSTAGE\F3FMjs2\webapps\servlet\ WEB-INF\classes\ 

    [Solaris]：
    /opt/FJSVjs2/webapps/servlet/WEB-INF/classes/

　

　

步驟五(Step 5)：重新啟動 Servlet Service 及 Web Server。

[Windows]：

  1. 停止 Web Server。

    在 NT 服務 視窗中停止 FJapache 服務。

  2. 停止 Servlet Service。

    在 DOS BOX 下切換目錄至 C:\INTERSTAGE\F3FMjs2\bin\
    執行 jssvstop 。

  3. 啟動 Servlet Service。

    在 DOS BOX 下切換目錄至 C:\INTERSTAGE\F3FMjs2\bin\
    執行 jssvstart 。

  4. 啟動 Web Server。

    在 NT 服務 視窗中啟動 FJapache 服務。

[Solaris]：

  1. 重新啟動 Solaris。
  2. 啟動 Servlet Service。

    執行 /opt/FJSVjs2/bin/jssvstart
  3. 啟動 Web Server。

    執行 /opt/FJSVihs/bin/httpd

步驟六(Step 6)：測試 jLIVE™ middle ware。

  1. 開啟 IE ，在 URL 中輸入
    HTTP://localhost
    IE 將會出現 Interstage 範例程式畫面，表示 HTTP Server 服務正常 。
  2. 在 URL 中輸入
    HTTP://localhost/servlet/com.probe.jslive.ProbeServlet/
    IE 將會出現 jLIVE™ middle ware 的管理模組，輸入 ID Password 後，按選Server Configuration 。
  3. 檢視 Authorized ID 及 Authorized concurrent users 是否為所購買的授權數。
  4. 如果出現任何無法啟動或非預期的狀況，請將【Interstage 安裝路徑】【F3FMjs2】【log】目錄下的記錄檔傳至 service@probe.com.tw。

　

　

關於應用程式 deployment 

  1. 由 jLIVE Builder™ 所產生的執行模組包括 *.html 及以下 class 的所有檔案，可安置於 HTTP Server。
    複製至何目錄，須查詢 HTTP SERVER conf 檔。

    [Windows]：
    C:\INTERSTAGE\F3FMihs\conf\httpd.conf。

    [Solaris]：
    /opt/FJSVihs/conf/httpd.conf。

    開啟並尋找 DocumnetRoot 其後所記載的目錄即為 Web Server root directory。

  2. HTTP server 及Interstage 預設是運作於 port 80，所以請檢查應用程式開發時的 middle ware HTTP 是否已改為 80 port。

關於 JDBC driver 

  1. 若只使用於 jLIVE™ 系統，可以直接將 JDBC jar 檔，複製於[Interstage 安裝路徑]\F3FMjs2\webapps\servlet\WEB-INF\lib 中，若是 class 檔則複製至[Interstage 安裝路徑]\F3FMjs2\webapps\servlet\WEB-INF\classes 中。
    例如：
    安置 middle ware 中的 oracle driver ：將     [jslive 授權光碟]\jslive\jre141\oracle_driver 目錄下的所有檔案複製至 【Interstage  安裝路徑】【F3FMjs2】【webapps】【servlet】【WEB-INF】【classes】目錄下。
