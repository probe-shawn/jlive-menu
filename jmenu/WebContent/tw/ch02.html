應用程式執行環境與機制 (Application environment and mechanism)

　

jLIVE Builder™ 所開發的網際網路應用程式，可以以下列三種方式執行：

  • 由瀏覽器啟動執行 (browser enabled HTML)

  • Java™ 可執行檔獨立執行 (standalone executable JAR)

  • 以 Java web start™ 啟動執行 (web start JNLP)

在網際網路上啟動執行後，應用程式皆以多層式的架構 (n-tiers) 運作。此一機制與過去的主從架構 (client-server) ，不論在內部的系統、資料流程或執行的作業平台，都是截然不同的。

以下章節將說明 jLIVE Builder™ 網際網路應用程式在企業應用整合 (Enterprise Application Integration) 上的應用； jLIVE Builder™ 所開發的網際網路應用程式 (internet application) 與一般網頁 (web page) 在客戶端 (client) 、伺服器端 (server) 之差異與可行性； jLIVE Builder™ 完整的開發環境與傳統程式設計環境 (traditional coding IDE) 的比較；網際網路應用程式 (internet application) 運作執行的程序與機制、以及如何以 jLIVE Builder™ 快速地開發網際網路應用程式 (internet application) ，架構企業應用的網際網路 n-tiers 環境 (web environment) 。

瀏覽器啟動執行 (browser enabled HTML)

jLIVE Builder™ 在應用程式開發完成，產生執行模組時 (build) ，會自動產生一個 HTML 及 Java™ Applet  原始檔案。佈建程式後，使用者只要在瀏覽器執行 HTML 檔即可執行程式。一個應用系統只會存在一個 Java™ Applet ， Applet 的作用只在於初始化瀏覽器的應用程式，應用程式接下來的所有運作，將會在此 Applet 區域，運用 Java™ J2SDK Swing set ，以多視窗模組的型式呈現。這也是為什麼 jLIVE Builder™ 的應用程式執行時不用不斷的換頁，以及為什麼 jLIVE Builder™ 的應用程式具有使用者及系統流程 (system  flow) 的原因。

以瀏覽器啟動執行的程式，使用者端 (client) 會根據執行模組的需求，自動下載及更新最新版的 Java™  class 模組，應用程式管理者只要把新增或更新的 Java™ class 模組，置入應用伺服器，所有使用者端 (client)  會自動更新。在執行環境上以瀏覽器啟動執行的程式，具有最佳的更新及分散載入的機制。

Java™ 可執行檔獨立執行 (standalone executable JAR)

　

jLIVE Builder™ 在應用程式開發完成，產生執行模組時 (build) ，系統會將所有 Java™ class 模組及資源，壓縮成一個可單獨執行的 Java™ 可執行 JAR 檔，應用伺服器佈建後，使用者下載此 Java™ 可執行 JAR  檔，即可在任何平台像執行一般 Windows 的程式一樣執行多層式網際網路應用程式。此種執行方式與一般 (client/server) 程式一樣，使用者須自行下載更新版本。

　

Java web start™ 啟動執行 (web start JNLP)

　

jLIVE Builder™ 在應用程式開發完成，產生執行模組時 (build) ，系統會產生 Java web start™ 啟動執行的 JNLP (Java™ Network Launching Protocol) ，佈建程式後，使用者只要啟動 Java web start™ 即可執行程式。程式執行時不需經過瀏覽器，程式執行的外觀及方式與一般 Windows 的程式一樣，與單獨執行的 Java™ 可執行 JAR 檔不一樣的地方是， Java web start™ 會控制版本更新，在應用程式管理者更新應用伺服器程式後，使用者端的 Java web start™ 會自動載入新版。
