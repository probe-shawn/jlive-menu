應用程式

jLIVE 應用程式模組包含 資料庫 視窗畫面 報表 及 執行程式，按下新增鍵後，在控制面板中將會出現應用程式模組以及個各節點•新增應用程式，必須先開啟 jLIVE middle ware並指明其所在的URL，可以按連結測試鍵，測試是否連結成功•
新增應用程式 :  新建立應用程式
測試 middle ware 連結 :  檢查 middle ware 是否已啟動，URL是否設定正確
取消 :  放棄新增
存檔 :  任何的設定修改後必須存檔
刪除 :  刪除應用程式，系統將會移除控制面板上的節點，相關的檔案資料不受影響，不會被刪除
專案名稱 :  jLIVE 可以在一個專案中，產生多個執行方式不一樣的程式
Http :  jLIVE middle ware URL
登入 :  是否在使用者登入時，開啟密碼驗證視窗，使用此項功能必須配合執行程式模組中的安全控管機制
作者 :  作者
版本 :  版本
路徑 :  此專案應用程式的預設路徑 
存取鍵 :  工作群組存取本機資源的授權碼，可以是任何字串值
群組 :  按購買的jLIVE系統授權數而定，在第一欄位中填入工作群組的電腦名稱或IP，第二欄位填入該機的存取鍵值，執行工作群組掃描即可分享同一專案名稱下的設計模組
描述 :  應用程式備註
