Form
jLIVE Builder follow the object oriental programming and event-driven concept to make applications. First, specify the data source that will be manipulated within the form;  layout data-aware components to control data flow;  route the user interactive events to complete process logic; then set the form properties and let it be called by other form or menu.
Create new form :  Create a new form for design.
Cancel :  Cancel.
Save :  Save the modified properties.
Delete :  Remove this form.
Make Java :  From XML to generate Java source code.
View Java :  Java source code viewer.
Compile Java :  Compile Java source to class.
Build :  From XML to generate Java source then compile it to class.
Class Name :  The Form name, cannot be blank, and must follow the Java variable naming convention to name it. System will generate Java class base on this name, and the application will use the name as reference of this form. Change the existed form name, system will check name duplicated or not, then alter all the files which depend on it.
Author :  Author.
Version :  Version.
Category :  The category of form in this application, used by control panel to hide or show it on the control tree node.
XML File :  The form properties xml file. System will generate a new one when first time enter the form design mode.
Java Source :  The system generated Java source code.
Class Path :  The default path for Java compiler to output class file.
Description :  Description.
