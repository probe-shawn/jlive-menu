執行 NON_SELECT SQL 命令 (execute NON_SELECT SQL) 

　

事件執行狀態與訊息 (event runtime status and message)

　

執行非 SELECT 的SQL 語法，例如：INSERT、DELETE、UPDATE 等 SQL 指令集 。在設計模式下，如果資料庫系統支援 Roll Back，則在 "測試 SQL 命令" 時，資料庫系統會回復 (roll back) SQL 執行的結果，而不會真正改變資料。
　

例如，輸入：
    DELETE FROM orderdetails WHERE OrderID=?
      參數類別INTEGER，運算式 {orderid}，按右方 SQL ，會出現訊息盒，告知影響列數 (rows)，並不會真正刪除資料來源 orderdetails 的資料。然而在 測試執行模式 (run mode) 時，執行這個事件，則會真正刪除資料。

  • 資料庫 (database) ： 選擇資料庫別名。
    　

  • Non_select SQL 命令 ： 非 Select 語法 SQL 語言編譯區。輸入 SQL 指令，可以按下 F2 鍵，SQL 資料欄位輸入輔助視窗會出現，設計師可以按選資料欄位，加快指令輸入的速度及正確性。 
    　

  • 參數 (parameters)： 
    　

    • 類別 (type)： 元件型態。選擇字串、數字、日期、時間、邏輯...等。　

    • 運算式 (formula)： 雙擊儲存格輸入運算式。
      　

  • SQL Error Handle ： 當發生 SQL 錯誤時，可以事件承接這個錯誤訊息， 再由設計師控制其後續的處理 。(參閱 SQL error handle 的說明)

事件執行狀態與訊息 (event runtime status and message)

　

EvStatus("事件名稱") ：取得事件執行狀態。
　

回傳值 狀態
空字串 事件未被引發。
-1 事件啟動後，依條件判斷後執行否則事件。
1 事件啟動後，執行成功。
0 事件啟動後，執行過程有誤。

　

EvStatus("事件名稱.error") ：取得事件執行錯誤代碼。

　

回傳值 說明
空字串 或 0 沒有錯誤訊息或無法取得錯誤訊息。
1 資料庫 SQL 執行有誤。

　

EvStatus("事件名稱.INFO_STRING") ：取得事件執行 INFO_STRING 的訊息。

　

NON_SELECT SQL 的執行結果，統一由 SQL_ERROR Handle 處理。請參閱 SQL error handle 的說明。
