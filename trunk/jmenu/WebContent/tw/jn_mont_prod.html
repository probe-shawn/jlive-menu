【Northwind】-- Sample Application

　

Sales Information by Month and Products

　

視窗名稱 (Form Name)：MONT_PROD
　 　
操作說明 (Specifications)：
　

  • 單項產品當年度每個月份的訂單數量、銷售金額、單張訂單平均銷售額資料。
資料來源 (Data Source)：
Product：

產品資料，作為 combobox 關連的資料來源。
【SQL】：Select ProductID,ProductName  From products
Monthly_Sales：

SQL資料來源。
【SQL】：
   select sum(A.UnitPrice*A.Quantity*(1-A.Discount))

          as subtot,
          Month(B.orderdate) as orderdate,

          sum(1) as orders
   From   orderdetails A left outer join orders B
          on A.orderid=B.orderid 
          left outer join products C
          on A.productid=C.productid
   where  A.productid=? 
   group  by orderdate 
   order  by orderdate
       參數類別 "INTEGER"，運算式為  {combo_Name}，查詢產品銷售資料。
元件 (Components)：
【combo_Name】

下拉式清單方塊元件 (ComboBox)。
【關連資料來源】【資料欄位】連接 Product / ProductID (產品編號)。
【被選時移至關連資料來源記錄】核取此項。
【顯示】運算式為 {Product:ProductName} (產品名稱)。
【關連資料來源初始化時清除選擇】核取此項，則視窗開啟時，此元件為空值。
【btn_trig】

按鈕元件 (button)。
【動作事件】選擇 trig_month ，按下時觸發 "Monthly_Sales" 資料來源 。

【grid_sales】

資料表元件 (Grid)。
【資料來源】連接 Monthly_Sales。
【欄位】【資料表元件欄位】 的最後一個欄位 "Avg. per order" 為計算欄位，運算式為
   if   {Monthly_Sales:orders} <> 0

   then {Monthly_Sales:subtot}/{Monthly_Sales:orders}

   else 0

       如果該月訂單數量 > 0，則計算單張訂單平均金額，否則填 0。

事件與事件表 (event schema)：
【trig_month】

觸發資料來源事件。
【執行】【資料來源】選 Monthly_Sales。
【SQL】為：
   select sum(A.UnitPrice*A.Quantity*(1-A.Discount))

          as subtot,
          Month(B.orderdate) as orderdate,

          sum(1) as orders
   From   orderdetails A left outer join orders B
          on A.orderid=B.orderid 
          left outer join products C
          on A.productid=C.productid
   where  A.productid=? 
   group  by orderdate 
   order  by orderdate
        參數類別 "INTEGER"，運算式為  {combo_Name}，依所選擇的產品，

        重新查詢其銷售資料。
