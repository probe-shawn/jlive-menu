Applet

系統將會自動產生一個 HTML 檔，內含一個 Java applet 。應用程式的使用者可以用瀏覽器執行此應用程式。所有設計的視窗及報表只要安裝於 Web ，使用者執行到該程式模組時才由 Web 送出，且只要該模組沒有任何更新版本，使用者端即以本機上的模組直接執行不用再下載。此一機制不但執行效率高而且程式維護安裝容易。
JarRun

系統會將所有設計的視窗及報表，壓縮成一個可以單獨執行 Java Jar file。系統會自動產生可執行的主視窗框架，並由設計者選定初始的主畫面。使用者可像執行任何 Windows 的方式執行此應用程式。
Applet
建立 Applet run :  建立一個HTML，在 HTML 嵌入embedded Java Applet 小程式。
存檔 :  存檔。
安全控管 :  jLIVE 系統預設的安全登入機制。
應用程式名稱 :  一個專案應用程式可產生多個 HTML ，其中可設定不同的主視窗及權限，按使用者的需求登入不同的 HTML。
HTML 名稱 :  HTML 名稱，所產生的 HTML除了 APPLET 的標籤外，設計者可按需求作進一步的編輯。
Applet 名稱 :  系統會產生預設的 Applet Java 原始資料檔，設計者可按需求作進一步的編輯。
主視窗 :  應用程式執行的第一個視窗畫面，如果該視窗的屬性是主視窗，則視窗畫面將會崁入瀏覽器的畫面，否則以內部框架的型態呈現。
JRE Plugin :  此段 URL 敘述將內崁入 HTML 中，當使用者端沒有安裝 Java Runtime Environment 時瀏覽器將據此下載 JRE。此項空白則以預設 Sun 的 Web 為下載驅。
客製 Jar 檔 :  jLIVE 允許設計者呼叫外部 class，或自行開發的模組，必須在此填入所參照的 class 檔，在程式啟動時 jLIVE 會一併傳送給使用端。
敘述 :  設計者對於應用程式相關說明。
