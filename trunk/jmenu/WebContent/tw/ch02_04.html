備註：

  1. •jLIVE Builder™ 所開發的應用程式及中間層是100% pure java，可跨所有 OS 平台執行 。
    •ASP，VB/Delphi 只能運作於 Microsoft 作業平台。
  2. •jLIVE Builder™ 預設為最少 3-tier 架構。
    •ASP/JSP/PHP 及JBuilder/VisualAge 須自行尋找解決方案，並開發建置。
    •VB/Delphi 為 Client/Server 2-tier 工具。
  3. •jLIVE Builder™ 應用程式是分散式執行模組，使用者只下載當下執行所需的小執行檔 (class)，並非將整個應用程式下載至使用者端。
    •VB/Delphi 須下載全部應用程式。
    •JBuilder/VisualAge須自行尋找解決方案，並開發建置。
  4. •jLIVE Builder™ 資料庫伺服器位於防火牆內的第 3 tier，並可同時存取各種不同的資料庫系統。
    •其餘開發平台須自行尋找解決方案，並開發建置。
  5. •jLIVE Builder™ 在瀏覽器中以多視窗系統，服務使用者操作應用程式。
    •ASP/JSP/PHP 以不斷換頁的方式提供作業服務，無法管控操作流程。
  6. •jLIVE Builder™ 使用 Java class 在使用者端運作，使用者將具備有運算能力。
    •ASP/JSP/PHP 使用者只能透過瀏覽器瀏覽資料，計算程序須由 Web 擔負。
  7. •jLIVE Builder™ 報表系統以 inch 量測列印位置，列印時以紙張左上角絕對位置定位。
  8. •jLIVE Builder™ 以分散式計算，由資料庫伺服器運算列印資料，使用者端計算彙整並配置列印，提供列印服務。
  9. •jLIVE Builder™ 使用者權限資料儲存於資料庫伺服器。
    •其餘開發平台須自行尋找解決方案，並開發建置。
  10. •jLIVE Builder™ 以完全視覺化物件，提供設計者更新、拓展、經驗傳承應用程式。
    •其餘開發平台須以傳統式撰寫程式碼方式進行。
  11. •jLIVE Builder™ 是分散式系統，程式更新時只要上傳更新模組至Web即可。
    •VB/Delphi 及 JBuilder/VisualAge 須編譯且要求使用者端重新下載整個應用程式。
