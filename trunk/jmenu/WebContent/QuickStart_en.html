Quick Start

¡@
1. Install Java™ J2sdk

¡@

The current version of Java™ J2sdk is FCS 1.4.1_01, you can download from the java.sun.com url :

http://java.sun.com/j2se/1.4.1/download.html

Execute j2sdk-1_4_1_01-windows-i586.exe (for Windows) or j2sdk-1_4_1_01-linux-i586-rpm.bin (for Linux) and follow the installation guild to setup Java™ J2sdk environment.

¡@
2. Install jLIVE Builder™
¡@

Execute the Setup.bat (for Windows) or jSetup.jar (for Linux) and follow the installation guild to setup jLIVE Builder™.

The recommended installation path

Windows : c:\jLIVE\
Linux : /usr/jlive/

The license serial number 

Demo version : the serialno.txt located in the setup disk root directory/jlive/
Licensed version : the serialno.txt located in the licensed disk root directory/jslive/

After the installation, licensed version must copy the jLIVE/jalive.jar which come with licensed disk to the directory c:/jLIVE/jar/ and overwrite the same name file.

¡@
3. Before start the jLIVE Builder™

The third party collaborated softwares :

Apache Tomcat : jLIVE™  middle ware (jslive) container. 
The copyright notice come with the jLIVE Builder™ disk and located in c:\jLIVE\tomcat\LICENSE.

Apache James  SMTP (optional): for smtp mail testing only.
The copyright notice come with the jLIVE Builder™ disk and located in c:\jLIVE\james\LICENSE.

mySQL™ database (optional) : for demo program testing only.
download site -- http://www.mysql.com/downloads/index.html

mySQL™ Connector/J (optional) : JDBC driver for demo program testing only.
download site -- http://www.mysql.com/downloads/api-jdbc-stable.html

IDAutomationLinear Java Barcode Package (optional) : for testing and evaluation (trial version).
download site -- http://www.idautomation.com/java/

i-net JDBC driver (optional) : MSSQL™ database driver (trial version).
download site -- http://www.inetsoftware.de/English/produkte/JDBC2/Default.htm


Install mySQL™ database

  1. Download mysql-3.23.53-win.zip (setup zip file) from mySQL™ download site.
  2. Unzip mysql-3.23.53-win.zip and setup mySQL™ database to the directory c:\mysql\
  3. Copy the c:\jLIVE\mysql\data\northwind directory and include the following files to the c:\mysql\data\ directory.
  4. Copy the c:\jLIVE\mysql\data\mysql directory and include the following files to the c:\mysql\data\ directory and overwrite it.


Setup Connector/J mySQL™ JDBC driver

  1. Download mysql-connector-java-2.0.14.zip (driver zip file) from mySQL™ download site.
  2. Unzip mysql-connector-java-2.0.14.zip.
  3. Copy the mysql-connector-java-2.0.14-bin.jar to the directory c:\jLIVE\tomcat\lib\

Setup IDAutomationLinear Java Barcode Package 

  1. Download JavaBarCodeDemo.zip (package zip file) from IDAutomation download site.
  2. Unzip JavaBarCodeDemo.zip.
  3. Copy the LinearBarCode.jar to the directory c:\jLIVE\jar\

Setup i-net MSSQL™ JDBC driver

  1. Download Sprinta2000_Trial.zip (driver zip file) from inetsoftware.de download site.
  2. Unzip Sprinta2000_Trial.zip.
  3. Copy the Sprinta2000.jar to the directory c:\jLIVE\tomcat\lib\
4. Setting environment variables (JAVA_HOME and CATALINA_HOME)
¡@

Windows NT/2000/XP

  1. [Control panel][System][Advanced]-->Environment variable.
  2. Insert two new variables
    name : JAVA_HOME    value : c:\j2sdk1.4.1_01
    name : CATALINA_HOME value : c:\jLIVE\tomcat

Windows 95/98/me

  1. Edit the c:\autoexec.bat
  2. Add SET CATALINA_HOME=C:\jLIVE\tomcat and SET JAVA_HOME=C:\j2sdk1.4.1_01
  3. Restart Windows.
5. Startup mySQL™ database
¡@
  1. Execute c:\mysql\bin\winmysqladmin.exe.
  2. User name : PROBE and Password : PROBE for the first time enter the  WinMySQLadmin.
6. Startup Tomcat service (jLIVE middle ware)
¡@

Execute c:\jLIVE\tomcat\startup.bat.

¡@
7. Execute sample program
¡@

Executed in jLIVE Builder™ design platform.

  1. Startup jLIVE Builder™ : execute c:\jLIVE\exejdlive.bat.
  2. Expand NorthWind_sample application node : there are  [Database] [Form] and [Execute] three child nodes.
  3. Edit sample main panel : expand [Form] node, choose the form node named 'MAIN_MENU', click  right mouse button, select Design function, enter to design mode.
  4. Execute sample main panel : in the design mode, select the [Form][Run] menu item to run it.

Executed by Browser

  1. Double clicked the C:\jLIVE\programs\jNorth\jNorth.html .

Trouble Shooting
¡@
  • Check the j2sdk was properly installed.
  • Windows 95/98/me must restart after installation.
  • Tomcat
    • JAVA_HOME and CATALINA_HOME were set correctly.
    • An "out of environment space" error when running the batch files in
      Win9X/ME-based operating systems.
      Right-click on the STARTUP.BAT and SHUTDOWN.BAT files. Click on
      "Properties" then on the "Memory" tab. For the "Initial environment" field,
      enter in something like 4096.
      After you click apply, Windows will create shortcuts in the directory with
      which you can use to start and stop the container.
    • The most common hiccup is when another web server (or any process for that
      matter) has laid claim to port 8080. This is the default HTTP port that
      Tomcat attempts to bind to at startup. To change this, open the file:
      $CATALINA_HOME/conf/server.xml
      ...and search for '8080'. Change it to a port that isn't in use, and is
      greater than 1024, as ports less than or equal to 1024 require superuser
      access to bind to.
      Restart Tomcat and you're in business. Be sure that you replace the "8080"
      in the URL you're using to access Tomcat. For example, if you change the
      port to 1977, you would request the URL http://localhost:1977/.
