Applet

システムは自動的にJava Applet 込みのHTMLを生成する。ユーザはブラウザを使って、このアプリケーションを実行することができる。全てのフォームやリポートはWEBサーバーに保存し、必要なとき、必要なモジュールだけユーザに送る。もし、モジュールが更新されていなければ、ユーザのクライアントにあるモジュールを直接使用し、再度Downloadする必要がない。この構成はパフォーマンスが非常に効率的だけでなく、プログラムの保守も簡単に行うことができる。 

JarRun

システムは設計した全てのフォームやリポートをStandaloneに実行することのできるJava Jarファイルに圧縮する。システムは自動的にメインフォームフレームを作成し、設計者によって、初期メイン画面を指定することができる。ユーザは一般的なウィンドウアプリケーションと同じように実行することができる。

Applet
Applet 実行 :  Embedded Java Applet 込みの HTMLファイルを生成する。
保存 :  プロパディを保存する。
セキュリティコントロール :  jLIVE デフォルトセキュリティコントロールモジュールを使用する。
アプリケーション名 :  一つのアプリケーションには、幾つかのHTMLを生成することができる。それぞれのHTMLには異なるメインフォームとアクセス権を設定することができ、ユーザによって使い分けることができる。
HTML名 :  HTML の名前。生成されたHTMLの中、AppletのTAG以外、必要なニーズによって、設計者は拡張することができる。
Applet名 :  システムは自動生成するApplet　Java™ソースファイル。必要なニーズによって、設計者は拡張することができる。
メインパネル :  アプリケーションが実行されるとき最初のフォーム、ブラウザの表示領域に貼りこめる。
JRE Plugin :  Java Runtime Environmentがインストールされていなかったら、Java™ Runtime Environment Download先を指定する。デフォルトはSUN Micro 社のWEBサーバーを指定されている。
カスタムJarファイル :  jLIVE™では、プログラマーは外部クラスあるいは自作クラスを参照することができる。参照するクラスをここで指定する。プログラムが実行されたとき、ユーザ端へ配送される。
記述 :  アプリケーション相関情報を記入する。
