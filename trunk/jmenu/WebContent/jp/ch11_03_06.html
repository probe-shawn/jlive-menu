ストアドプロシージャ (execute stored procedure)

　

ストアドプロシージャパラメーターの転送 (the parameters of stored procedure)

　

  • データベース (database) ： データベース名の選択。

  • ストアドプロシージャ (stored procedure) ： ストアドプロシージャの選択。

  • パラメーター (parameters) ：

    • 名前 (name)： パラメーター名。

    • IO ： パラメーターはイン (in)、アウト (out) 或いはインアウト (inout)。(フォーミュラ GetSpOutParam(spname.paramname)の説明を参考)。

    • タイプ (type)： パラメーター状態(INTEGER、VARCHAR...etc)。

    • 精度 (precision) ： 精度。

    • 長さ (length) ： パラメーター長さ。

    • 小数 (scale) ： 小数。

    • フォーミュラ (formula)： ダブルクリック (double clicked) でフォーミュラパラメーターを入力します。

  • SQL Error Handle ： SQL エラーが生じる時，イベントでエラーメッセージを処理します 。 SQL Error Handle の説明参考)

　

　

ストアドプロシージャパラメーターの転送 (the parameters of stored procedure)

　

データベースによってストアドプロシージャのパラメーターIO 状態が違ってきます，jLIVE Builer™ は(in)、(out) 、 (inout) をサポートします。フォーミュラでパラメーターを転入し，フォーミュラ  GetSpOutParam("ストアドプロシージャ.パラメーター名") で取り出します。プロシージャデザインするときに， SQL Exception かエラーコード表示の方式で，先にストアドプロシージャを正確に実行できたかどうかを検証しなければなりません。 GetSqlErrcode("データベース別名.ストアドプロシージャ") でエラーコード入手するか， SQL Error Handle で SQL エラー (exception) を処理するかを決めます 。
　

GetSpOutParam("ストアドプロシージャ.パラメーター名")の場合，大文字小文字に区別がつきますので (case sensitive)，同じでなければなりません。例えば：GetSpOutParam("ストアドプロシージャ.@RETCODE") 。
　

jLIVE Builder™ は現時点で結果カーサーをサポートしません (result set cursor)。
