データベース(Database)

jLIVE システムはjLIVE ミドルウェアを通して、データベースと接続します。jLIVE ミドルウェアはJava　JDBC接続によって、データベースにアクセスします。jLIVE ミドルウェアはJDBCドライバーのサポート機能によって、内部作業を自動的に行います。アプリケーションは同時に幾つかの異なるデータベースをアクセスすることができます。データベースシステムを設計するとき、それぞれのデータベースはデータベースアリアスで識別され、参照されます。データベースシステムを設計するとき、データベースの特殊機能を使っていない限り、プログラムを修正なしで、簡単に他のデータベースへ移行巣rことができます。

データベース新規作成 :  新しいデータベースを作成します。  
データベース接続テスト :  ドライバーとURLの設定により、ミドルウェア通してデータベース接続をテストします。
キャンセル :  新規作成を取り消します。
保存 :  変更を保存します。
削除 :  データベースレファレンス【参照】を削除します。
データベース　アリアス :  データベースアリアスはプログラムデザインを行うとき、データベースのレファレンスとして参照されます。データベースアリアスは空白にしてはなりません。英文字を使用してください。なお、アプリケーション中、データベースアリアスは重複してはなりません。
ドライバー : 

データベースのJDBCドライバーのことを表します。コンボボックスからよく使用するドライバーをセレクトすることができます。システムは自動的に関連するURLデータをURLパターン中に書き込みます。ホストとデータベース名を修正すればいいです。もちろん、コンボボックスにはないドライバーを手動で記入することもできます。ミドルウェアを起動する時に、ドライバーを正しくインストールして置けなければなりません。詳しくはjLIVE ミドルウェアシステムを参考してください。

jLIVE  ミドルウェアは JDBC TYPE1, TYPE2, TYPE3, TYPE4 のドライバーをサポートしています。
URL : 

ドライバーを提供しているベンダーによって、URLの設定方式及びフォーマットが異なっています。それぞれのベンダーのマニュアルに従って設定してください。

例えば：
  • com.inet.tds.TdsDriveを選択します。
  • URL 欄は jdbc:inetdae:hostname:1433?database=DATABASE を表示します,  hostnameとDATABASEは編こする必要があります。
  • ホストは SUPER とし、データベースは DBとします。
  • URLはこのように、jdbc:inetdae:SUPER:1433?database=DB となります。
ユーザ :  データベースへログインするユーザ名です。
パスワード :  データベースへログインするユーザのパスワードです。
スキーマ(Schema) :  データベース　スキーマ(Database　Schema)。スキーマ(Schema)サポートしているデータベースシステムを使用しているときのみ、設定が必要です。例えば、Oracleデータベース。
タイプ :  JDBC データベースがSQLステートメントを実行するときのResult Setのタイプを表します。JDBC version 1.0 はTypeと concurrencyをサポートしていません。ドライバー自身がサポートしている以外は、FORWARD_ONLYを使用しないでください。システムパフォーマンスを低下させかねません。SCROLL_INSENITIVE+1はベンダー独自の強化機能です。例えば inet for mssqlはその例です。
Concurrency :  JDBCデータベースがSQLステートメントを実行するときのResult Setに同期アクセスステータスを表します。JDBC 1.0 と旧バージョンのドライバーは大抵UPDATABLEをサポートしていません。 JLIVE™ミドルウェアは必要な時、自動的にデータレコード更新を行います。UPDATABLE+1 とUPDATABLE+2 ベンダー独自の強化機能です。
記述 :  データベースの相関情報をプログラマーが記入します。
