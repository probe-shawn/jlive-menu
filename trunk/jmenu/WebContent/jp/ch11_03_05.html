NON_SELECT SQL コマンドの実行 (execute NON_SELECT SQL) 

　

ナン SELECT のSQL，例えば：INSERT、DELETE、UPDATE などの SQL コマンド集 。設計モジュールで，データベースは Roll Backをサポートすれば， "SQL コマンドテスト"をクリックした時，データベースは (roll back) SQL 実行の結果を回復できます。
　

例えば，入力：
    DELETE FROM orderdetails WHERE OrderID=?
      パラメーターがINTEGER，フォーミュラ {orderid}，右の SQLをクリックすると ，メッセージボックスが表示され，影響する行数 (rows)が告げられ，データソース orderdetails のデータが本当に削除されません。しかし，モジュール実行 (run mode)テストを実行する時，このイベントを実行すると，データが根本的に削除されます。

  • データベース (database) ： データベース名。

  • Non_select SQL コマンド ： ナン Select SQL区域。　

  • パラメーター (parameters)： 

    • タイプ (type)： コンポーネントスタイル。文字列、数字、日付、時間、ロジック...など。　

    • フォーミュラ (formula)： 保存枠をダブルクリックし，フォーミュラを入力。

  • SQL Error Handle ： SQL エラーが生じた時，イベントでメッセージを表示し，引き続きのイベントを実行 。( SQL error handle 参考)
