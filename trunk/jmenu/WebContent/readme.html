Introduction


Thank you for using this release of the jLIVE Builder™. 

jLIVE Builder™

Solution for developing, deploying and managing n-tier internet and intranet application, support report system for printing anywhere on any platform. 

A system for both the development and deployment of intranet and internet business applications, generally built over existing databases. It is not just another IDE. jLIVE Builder™ gives you a way to quickly make interactive database application for Java™ and deploy them over the Web -- all in one comprehensive package.

Features

jLIVE Builder™ is a brand new concept for system developers to accelerate creating a robust business application run on any platform via internet. 

  • The total solution for developing web application

The RAD (Rapid Application Development) environment with fully visual framework, make the computing technologies easily accessible for productive developer use. High-performance java swing components with MVC (Model, View, controller) internal data-aware architecture, has proven to be highly capable for helping developers adapt to and exploit advances in Event-Driven technology.
The easy and flexible deployment for running your application on browser or standalone executable jarfile.

  • N-tiers architecture, pure java middleware

Suitable for any web server which supports JVM, not restricted in MS Windows NT environment. Application server (middleware) service all kinds of client application and with loading balance to make significant improvements the throughput of data flow. 

  • Distributed computing with just in time formula interpreter

jLIVE Builder™ includes both web computing and client computing technologies. The run time client site module (browser or workstation) embedded a formula interpret that can just in time compute the user manipulations and response immediately. Some like verify user's input , counting data relative with database fields content and user's input, and the chart data generated, will perform in client site. That will result in high response performance and avoid web server collapse due to massive computing services.

  • The real time user interactive database operation

jLIVE Builder™ process the user SQL request that invoke by visible components and communicate with middleware to obtain the result. The middleware connect and process all the database operations by JDBC driver, retrieve result sets then sent back to the requested client.

  • The Event-Driven, object-oriental programming

Easy deploy, no matter what web database usage, The developer can focus all efforts on domain know-how implements to create robust business applications.

  • Probe report system

jLIVE Builder™ report system is highly integrated with FORM design frame work , it support the same visual RAD environment as FORM design, drag and drop printable object to layout report. The printable content is dynamic computed by jLIVE™ formula, so all the components content and data source can be printed. The report data source is dynamically provided by application server, and computing by client site. this distributed computing mechanism will make the report system more robust and efficiency.

System Requirements

Windows 95, Windows 98, Windows NT 4.0, Windows 2000 or Window XP operation systems running on Intel hardware. A Pentium 166 MHz or faster processor.

At least 64 megabytes of physical RAM is required to run GUI applications. Forty-eight megabytes is recommended for applets running within a browser using the Java Plug-in product. Running with less memory may cause disk swapping which has a severe effect on performance. Very large programs may require more RAM for adequate performance. 

You should have 150 megabytes of free disk space before attempting to install the Java™ 2 SDK software. If you also install the separated documentation download bundle, you need an additional 120 megabytes of free disk space.
