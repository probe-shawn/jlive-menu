Database

jLIVE system through the middle ware to connect the database. The middle ware use the Java JDBC to connect and control the database. Due to the third party JDBC driver is not fully support the JDBC 2.0 so jLIVE middle ware will adjust the operations to fit the client request. The application can have multiple databases manipulation. The jLIVE internal programming which refer to the database by logical name that were so called DB Alias,  if do not use the special database dependent functions, the application can change the database, event no code changed necessarily.
Create new database :  Create a new database.
Test connection :  Through the middle ware, use the driver and URL to connect the database.
Cancel :  Cancel.
Save update :  Save the modified properties.
Delete :  Remove this reference.
DB Alias :  The reference name use in programming. Can not be blank and must follow the Java variable naming rule.
Driver :  JDBC database driver. You can select the driver from the combo box, system will fill the URL pattern in the URL box, just modified the host and database will do. or you can specify the driver which not in combo box by yourself. Beware that jLIVE middle ware must boot first and the driver must install  properly.  jLIVE middle ware support TYPE 1, 2, 3, 4 JDBC driver.
Url :  URL pattern. must refer to the driver document which provided by driver vendor. for example : 
  1. Select the driver  com.inet.tds.TdsDrive.
  2. URL box display jdbc:inetdae:hostname:1433?database=DATABASE , The hostname and DATABASE must be modified.
  3. Supposed the host is SUPER and the database is DB
  4. Then change it to jdbc:inetdae:SUPER:1433?database=DB
User :  The database user log in name.
Password :  The database user log in password.
Schema :  The Schema of database.
Type :  The SQL result set type, which request when prepared the SQL statement. JDBC 1.0 does not support Type and Concurrency. Do not use the FORWARD_ONLY if you have another chose, because the type will decrease the system performance. The SCROLL_INSENITIVE+1 is enhanced by driver vendor. for example : inet for mssql
Concurrency :  The SQL result set type, which request when prepared the SQL statement. The JDBC 1.0 and most driver are not support UPDATABLE. jLIVE middle ware will generate the update SQL when needed.UPDATABLE+1 and UPDATABLE+2 are enhance by driver vendor individually.
