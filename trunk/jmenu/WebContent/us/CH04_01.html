Before you design your application, you have to create new Datasource. jLIVE Builder support any well-known and widely-used database. Such as ORACLE,MSSQL,DB2,SYBASE......

¡@

4.1 Create New DataSource

¡@

Please click the button " Create New DataSource "  which looks like below image to create datasource.

¡@

You can delete a database or save or test db connection. But please take a notice of that every time you change the setting, be sure to click save button.

  • DB Alias : The logical name of the database, It is user defined.

  • Driver : The database JDBC driver usually decided by the invaded third party company. You must choose particular driver which is designed to support for your database.

  • Url : JDBC URL, provide the method that driver can identify and access to the database. Different driver have different url format. The following examples  illustrate how to connect these well known databases. You have to input correct hostname/IP and DataBase name. If there is not any module formats fits you need, you can type a new one.

  • JDBC to connect mySQL

DB Alias : NorthWind
Driver : org.gjt.mm.mysql.Driver
Url : jdbc:mysql://HostName:3306/DatabaseName
¡@

  • JDBC to connect ORACLE

DB Alias : ORACLE
Driver : oracle.jdbc.driver.OracleDriver
Url : jdbc:oracle:thin:@HostName:1521:DatabaseName
¡@

  • iNet MSSQL2000 Driver 

DB Alias : MSINET
Driver : com.inet.tds.TdsDriver
Url : jdbc:inetdae:HostName:1433?database=DatabaseName&charset=Cp950

* Cp950 : Key in the char set (Options), for chinese please type Cp950 or Ms950 ( for Japanese please type JIS0212 )
¡@

  • ODBC to connect Access

DB Alias : ODBC_ACCESS
Driver : sun.jdbc.odbc.JdbcOdbcDriver
Url : jdbc:odbc:ACCESS_ODBC_Name

¡@

  • User : This is database login user name. It is necessary to create database user before you test connection with database.

  • Password : This is database login user name & password. It is necessary to create database user before you test connection with database.

  • Schema : It is effective when using ORACLE database. You have to key in  user schema which is a set of object owned by the user ID. This is not necessary to input schema name but if do so you can find the store procedure more quickly. Because ORACLE database beforehand store procedures might be hundreds. With the schema identified you will see the store procedure under schema.

  • Type : It is required to choose one type of database. You can find database type in their reference manual. The following example is MySQL database and it's DB type is " SCROLL_INSENSITIVE ".

  • Concurrency : It is required to choose one concurrency type of database. You can find database concurrency type in their reference manual. The following example is MySQL database and it's DB concurrency type is " READ_ONLY ".

  • Connection :

                              Shared : All client use same connection if check the check box.
